# Reaction Chain

> Interactive Knowledge-Gated Mechanic #2 — Sequential Process Mastery

**Players:** 1 | **Mode:** Solo | **Buzan Integrated** | **Bloom's:** Apply | **PISA:** 2-3 | **AI Tier:** Tier 1

---

## 1. Core Game Mechanics

Reaction Chain presents a **visual chain of 6-10 nodes** arranged in a connected sequence. Each node represents one step in a process — photosynthesis stages, historical event chains, algorithm execution flow, mathematical procedure steps, or biological cycles. The chain starts **unlit (dimmed/gray)**. Students answer subject questions to ignite each node in order.

- **Chain construction:** The chain of 6-10 nodes is generated incrementally. The first 2-3 nodes are visible from the start. Additional nodes appear only after the preceding node is successfully lit. This prevents students from gaming the full sequence and keeps cognitive load focused on the current step.
- **Each correct answer:** Adds/activates the next node in the chain. The node illuminates with a color-coded glow (see Buzan Integration) and a brief animation (pulse + connection line draw to the previous node). The process step label appears on the node.
- **3 wrong answers in a row:** Breaks the chain. All nodes from the break point onward dim again. The student must **rebuild from the break point** — previously lit nodes before the break remain lit, but the chain cannot advance until a correct answer is given at the break point.
- **Goal:** Complete the full chain — all nodes lit and connected. A completion animation traces the entire chain from start to finish.

**Chain length scales by grade:**

- **Grades 1-3:** 6 nodes, no time pressure, generous hints. Process topics: daily routines, simple cycles (water cycle basics), counting sequences.
- **Grades 4-6:** 7-8 nodes, moderate hints. Process topics: photosynthesis steps, arithmetic algorithm flow, story plot sequence.
- **Grades 7-9:** 8-9 nodes, minimal hints. Process topics: chemical reaction mechanisms, historical event causation chains, sorting algorithm steps.
- **Grades 10-11:** 10 nodes, no hints. Process topics: cellular respiration full pathway, multi-step proof sequences, complex historical causation chains.

**Break point recovery:** After a chain break, the next question at the break point is drawn from the same process step (not a new step). This ensures the student masters the current step before advancing. If the student breaks the chain a second time at the same step, the system offers a scaffolded hint (first letter, keyword highlight, or visual cue).

**Subject compatibility:** Any subject with sequential/ordered content. Best suited for: science (processes, cycles), history (event chains), math (algorithms, procedures), language (grammar transformation steps), informatics (algorithm flow, program execution).

## 3. Buzan Integration: Color-Coded Sequence Encoding

Reaction Chain leverages **Tony Buzan's color and imagery principles** to strengthen sequential memory and process comprehension.

- **Color Hooks (Synesthesia):** Each node in the chain is assigned a color based on its position in a rainbow spectrum (ROYGBIV). Node 1 = Red, Node 2 = Orange, Node 3 = Yellow, etc. This creates a natural color-ordered memory scaffold — students can recall "the green step" or "the purple step" even if they forget the formal label. The color gradient also provides an immediate visual progress indicator.
- **Radiant Thinking (Process Hub):** The completed chain is treated as a radiant structure. The first node (Red) is the "root" or starting condition. Each subsequent node branches from the previous one, with connection lines drawn as flowing energy paths. When the chain is complete, a brief animation pulses energy from the root through all nodes to the terminal node, reinforcing the causal flow of the process.
- **Imagery & Exaggeration:** Each node includes a small icon or visual metaphor for its process step (e.g., a lightbulb icon for "energy capture" in photosynthesis, a domino falling for "causation" in history chains). Bizarre or exaggerated imagery is preferred — per Buzan, unusual visuals are 3-7x more memorable than text alone. Icons persist in the student's process gallery for review.
- **Memory Palace Anchor (Chain Milestone — rare, ~5% of chains):** One node in the chain carries a icon. Lighting this node triggers a 5-second mini-palace sequence: a familiar location appears, and the process step is "placed" at one location within it. Students who complete all Palace Chains across sessions unlock the "Process Architect" achievement in the Bilim Bazasi.

## 4. Question Styles & Interaction Mechanics

Each node in the chain requires answering a subject question to activate. The question style scales with the PISA level of the content and the student's grade band.

| PISA Level | Question Style | Chain Context | Cognitive Target | Wrong Answer Feedback |
|-----------|---------------|--------------|-----------------|----------------------|
| **L1** | Direct recall: "What comes next after [step X]?" with visual cue | Simple daily routines, counting sequences, basic cycles | Direct recall — identify the next item in a familiar sequence | Visual hint appears (first letter, icon highlight). No chain break until 3 consecutive wrong. |
| **L2** | Identification with context: "In photosynthesis, after light is absorbed, what happens next?" | Standard processes: water cycle, story sequence, arithmetic steps | Property/step identification within a known process | Brief process context reminder. Wrong count increments toward chain break. |
| **L3** | Causal inference: "Why does step X lead to step Y? Choose the correct mechanism." | Multi-step processes: chemical reactions, algorithm flow, historical causation | Causal reasoning — understand why one step causes the next | Explanation of the causal link is shown after 2 wrong answers. Chain still breaks at 3. |
| **L4** | Process analysis: "If step X were removed, what would happen to the chain?" | Complex systems: ecosystem food chains, multi-stage industrial processes | Process analysis — predict downstream effects of changes | System shows the actual downstream effect of the wrong answer as feedback. Chain breaks at 3. |
| **L5** | Evaluation: "Which step in this chain is the weakest link? Justify." | Multi-variable processes: climate systems, economic cycles | Evaluation — assess process quality, identify bottlenecks or failure points | Counter-evidence presented. Chain breaks at 3, with full process analysis shown on rebuild. |
| **L6** | Synthesis: "Design an alternative chain that achieves the same outcome." | Open systems: evolutionary pathways, algorithm design, policy cascades | Synthesis — create alternative valid process sequences | Partial credit for valid alternative steps. Chain breaks at 3, but student's valid alternatives are saved. |

> **Interaction Model:** Questions appear in a panel below the chain. The student selects or types their answer. Correct answers trigger the node activation animation immediately. Wrong answers trigger a subtle shake of the current (unlit) node and increment the wrong counter. The chain itself remains visible throughout — the visual tension between lit and unlit nodes drives engagement.

## 5. Victory Conditions & Scoring

### A. Chain Victory (Standard)

Light all nodes in the chain without triggering a permanent break. Victory screen shows:

- **Unbroken Chain:** Zero chain breaks (never hit 3 wrong in a row). Full chain animation plays from root to terminal node. Bonus 200 XP.
- **Recovered Chain (1 break):** Broke once but rebuilt to completion. +100 XP bonus. The chain shows the break point with a scar marker.
- **Hard-Fought Chain (2+ breaks):** Standard XP, no bonus. Encouraging "Hali emas" framing for the break points, with full process review available.

### B. Chain Exhaustion (Defeat)

If the student repeatedly fails to light the final nodes (e.g., 5+ breaks at the same step, or the session timer expires):

- **Below 50% chain lit:** "Hali emas" screen. The full correct chain is shown for 5 seconds. Student must replay this chain in the next session (tracked as a pending mastery item).
- **50-99% chain lit:** Partial credit awarded. Remaining nodes are shown with correct answers. Student earns XP for lit nodes only. No forced replay — counted as a learning session, not a failure.

### C. Scoring Breakdown

| Event | XP | Notes |
|-------|----|-------|
| Correct node lit | +80 | Base reward per node |
| Unbroken Chain (0 breaks) | +200 | Completed entire chain without 3-wrong break |
| Recovered Chain (1 break) | +100 | Broke once, recovered to full completion |
| 3-correct streak within chain | +50 (Basic) / +75 (Premium) | Consecutive correct answers without any wrong |
| Chain break penalty | 0 | Nodes from break point dim; must rebuild. No XP loss for already-lit nodes. |
| Memory Palace node lit | +120 | Rare node (base + bonus) |
| Chain exhaustion (<50%) | 0 | Must replay; no XP awarded on failed chain |
| Partial chain (50-99%) | Per-node XP only | 80 XP x nodes lit, no completion bonus |

### D. Time Attack Mode (Premium)

In Time Attack mode, the student races to complete the chain as fast as possible:

- **Speed bonus:** Added XP based on completion time relative to the average for that chain length. Top 10% = +150 XP, top 25% = +100 XP, top 50% = +50 XP.
- **Chain breaks still apply:** 3 wrong in a row breaks the chain, adding rebuild time. The timer does not pause during rebuild.
- **Leaderboard:** Completion times are tracked per chain topic. Students can see their own best times and class rankings (opt-in).
