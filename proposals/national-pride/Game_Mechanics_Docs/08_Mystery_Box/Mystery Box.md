## 1. Core Game Mechanics

The Mystery Box mechanic operates in **two distinct modes**, both serving the same cognitive purpose: forcing the brain to recognize *which* tool to use, not just *how* to use it.

### Mode A: Session Phase Mystery Box (Structured)

Presented during Phase 3 Game Break as part of the dual-catalog selection. The student sees **3-5 closed boxes** on screen, each visually distinct (different colors, symbols, or shapes). Each box contains **one problem from a different subject category** (interleaved):

- **Step 1 — Identify:** The student selects a box. The box opens with an animation, revealing a problem. The student must first **identify the problem type** (e.g., "This is a fraction problem," "This is a geometry problem," "This is a reading comprehension problem").
- **Step 2 — Solve:** After identifying the category, the student solves the problem using the appropriate method.
- **Step 3 — Feedback:** The system evaluates both the category identification and the solution. Correct identification + correct solution = full XP. Correct identification + wrong solution = partial XP. Wrong identification = reduced XP with corrective feedback.

### Mode B: Random Case-Opening Event (Surprise)

At random milestones (session completions, streak achievements, level-ups), a Mystery Box **appears unexpectedly** as a case-opening event. This contains a **single challenging question** drawn from a broader difficulty pool than the structured mode. Every attempt yields XP — there is no zero-reward outcome.

**Interleaving Design (Rohrer & Taylor, 2007):** Problems are deliberately mixed across categories rather than blocked by type. This forces the student to engage in *discriminative contrast* — comparing problems to determine which strategy applies. Research shows interleaving improves transfer performance by +43% compared to blocked practice, because it trains the recognition step that blocked practice skips entirely.

**Box count scales by grade:**

- **Grades 1-3:** 3 boxes, 2 categories max, heavy visual cues on boxes (icons hint at content type).
- **Grades 4-6:** 4 boxes, 3-4 categories, subtle visual cues (color coding only, no explicit icons).
- **Grades 7-9:** 4-5 boxes, 4-5 categories, no visual cues — boxes are visually identical.
- **Grades 10-11:** 5 boxes, 5+ categories, no cues + problems may span cross-disciplinary boundaries.

**Category pool:** Boxes are drawn from the current session's subjects plus one "wildcard" category from a previously mastered topic (spaced interleaving). The wildcard ensures previously-learned material does not decay.

## 3. Buzan Integration: Curiosity & Sensory Anchoring

Mystery Box leverages **Tony Buzan's curiosity-driven learning** and **multi-sensory encoding** principles to maximize problem-type recognition and retention.

- **Color Hooks (Category Chromatics):** Each problem category is assigned a distinct color that is *hidden* during box selection but *revealed* when the box opens. The entire screen floods with the category's color for 0.5 seconds upon reveal, creating a strong chromatic anchor. Over repeated exposures, students begin to subconsciously associate box visual patterns with category colors, training implicit pattern recognition alongside explicit problem-type identification.
- **Sensory Anticipation (Curiosity Gap):** The closed-box state creates a deliberate information gap. Per Buzan's principle, curiosity is the brain's strongest encoding state. The moment between selection and reveal is maximized with a 1.5-second suspense animation (box rattling, light leaking from seams). This anticipatory arousal activates the locus coeruleus-norepinephrine system, enhancing memory consolidation for whatever problem appears inside.
- **Imagery & Exaggeration (Problem-Type Personification):** Each problem category has a recurring "character" that appears in the corner of the revealed problem. Fractions = a pie-splitting character, geometry = a shape-shifting creature, reading comprehension = a scroll-bearing scholar. These characters use exaggerated expressions matching the problem's difficulty (confused for tricky problems, excited for straightforward ones). Over time, the character becomes a retrieval cue for the problem-solving strategy.
- **Radiant Association (Cross-Link Web):** When a student correctly identifies a problem type AND solves it, a "radiant web" animation draws connections from this problem to 2-3 previously-solved problems of the same category that appeared in earlier sessions. This visually encodes the idea that problems are not isolated events but nodes in a growing knowledge network. Students who build webs connecting 10+ problems across sessions unlock the "Pattern Recognizer" achievement in Bilim Bazasi.

## 4. Question Styles & Interaction Mechanics

Mystery Box problems are drawn from the current session's content pool plus wildcard categories. The interaction follows a two-stage process: (1) identify the problem type, (2) solve it.

| PISA Level | Problem Type | Identification Stage | Solution Stage | Cognitive Target |
|-----------|-------------|---------------------|---------------|-----------------|
| **L3** | Standard curriculum problem from a recognizable category (e.g., "Solve 3/4 + 1/2") | Student selects from 3-4 visible category labels (e.g., "Addition," "Geometry," "Reading") | Standard solution input (numeric, text, or MC depending on subject) | Apply — recognize the operation, execute the procedure |
| **L3 (hard)** | Problem with surface features that mimic a different category (e.g., a word problem that looks like reading but requires math) | Student selects from 4-5 category labels; distractor categories are deliberately plausible | Standard solution input | Analyze — see through surface features to the underlying structure |
| **L4** | Multi-step problem requiring two concepts from different categories (e.g., calculate area then compare as a fraction) | Student identifies BOTH category labels (e.g., "Geometry + Fractions") — partial credit for identifying one | Multi-step solution with intermediate checkpoints | Apply/Analyze — decompose, map sub-problems to tools, execute in sequence |
| **L5** | Real-world scenario with ambiguous category boundaries (e.g., a bazaar pricing problem involving math, economics, and logic) | Student identifies the PRIMARY category and explains why (short text or MC rationale) | Open-response solution requiring justification, not just a numeric answer | Evaluate — determine which analytical lens is most appropriate, then apply it rigorously |
| **L6** | Cross-disciplinary problem with no single correct category (e.g., a historical data analysis requiring math + history + reasoning) | Student must identify ALL relevant categories AND rank them by importance | Open-response solution requiring synthesis of multiple disciplinary perspectives | Synthesis — integrate tools from multiple domains into a coherent solution strategy |

> **Identification Accuracy Tracking:** The system tracks not just whether the student solved the problem, but whether they correctly identified the problem type. A student who solves correctly but identifies wrong has a *lucky guess* profile — they may be pattern-matching without understanding. This triggers adaptive adjustment: future boxes will include more surface-mimic problems to force deeper analysis.

## 5. Victory Conditions & Scoring

### A. Session Phase Mystery Box Scoring

Each box is worth up to **250 XP**, split between identification and solution:

| Outcome | XP | Notes |
|---------|----|-------|
| Correct ID + Correct solution | 250 | Full reward — both recognition and execution mastered |
| Correct ID + Wrong solution | 125 | 50% XP — student knows WHAT to do but not HOW. Targeted feedback provided. |
| Wrong ID + Correct solution | 100 | 40% XP — lucky guess pattern. System flags for adaptive adjustment. |
| Wrong ID + Wrong solution | 50 | 20% XP — minimum participation reward. Full corrective feedback shown. |
| Perfect board (all boxes correct ID + correct solution) | +100 bonus | Added on top of individual box XP. "Master Classifier" badge for the session. |

### B. Random Case-Opening Event Scoring

Case-opening events use a **performance-based reward system** with guaranteed XP:

| Outcome | XP | Bonus Reward Pool |
|---------|----|------------------|
| Correct solution | 200 base | XP Boost 40%, Extra AI Usage 25%, Avatar Frame 15%, Customization Item 10%, Coupon 7%, Mythical Boss Event Ticket 3% |
| Wrong solution (attempt made) | 100 base | No bonus reward, but XP still awarded. Full solution walkthrough provided. |
| Challenging question correct (L5+) | 250 base | Same bonus pool + 2x Mythical Boss Ticket odds (6%) |

### C. Bonus Reward Pool Details

When a student solves a case-opening question correctly, they receive XP **plus one random bonus reward** from the pool:

- **XP Boost (40%):** +50% XP on all activities for the next 3 sessions.
- **Extra AI Usage (25%):** +3 AI assistant uses added to daily quota.
- **Avatar Frame (15%):** Cosmetic border for the student's avatar, themed to the subject category of the solved problem.
- **Customization Item (10%):** Unlockable skin, color, or accessory for the student's profile or game board.
- **Coupon (7%):** Redeemable for a hint skip, a boss redo, or a peer challenge invitation.
- **Mythical Boss Event Ticket (3%):** Entry to a rare, elite Mythical Boss challenge (cross-subject, PISA 6-level difficulty, exclusive rewards).

### D. Defeat Framing

Mystery Box has **no true defeat state** — every attempt yields XP. However:

- **All wrong (every box: wrong ID + wrong solution):** "Hali emas" (not yet) screen. The system shows all correct category identifications and provides worked solutions. The student is offered an optional "Retry Board" in the next session with the same boxes (re-shuffled order) for mastery credit.
- **Below 50% correct identification:** The system flags "Category Confusion" and adapts future box compositions to include more practice on the confused categories.
- **Below 50% correct solution (but good identification):** The system flags "Strategy Gap" — the student knows what tool to use but cannot execute it. Future sessions will include more scaffolded practice of the weak skill.
