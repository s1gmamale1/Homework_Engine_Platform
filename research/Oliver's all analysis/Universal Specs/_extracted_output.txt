# NETS Homework Engine — Blueprint Summary
Source: standards/NETS-Homework-Engine-Blueprint.docx
Revision: Incorporates IMPROVEMENTS_TO_THE_CURRENT_FRAMEWORKS.md (2026-04-07)
Relationship: UNIFIED = WHAT (content, standards, games). Blueprint = HOW (session lifecycle, engagement hooks, implementation). Neither sufficient alone.
Markers: [SPEC §X.X] = cross-ref to UNIFIED spec. [HOOK] = engagement mechanic. [AI: Tier X] = which AI handles it. [NEW] / [UPDATED] = changes from this revision.
Duration (UPDATED): Standard: 30-45 min (was 20-30) | Extended: 45-60 min (was 40-50) | Recovery: unchanged
# Engagement Philosophy
The Blueprint builds on four behavioral frameworks to turn the UNIFIED spec into an addictive (but ethical) learning loop:
White Hat gamification (epic meaning, accomplishment, empowerment) as primary motivator — chosen over Black Hat (scarcity, loss avoidance) which is used sparingly and only for re-engagement.
Hook Model (Nir Eyal): every phase has Trigger → Action → Variable Reward → Investment. External triggers (push notifications) convert to internal triggers (streak fear, progress desire) over time.
Octalysis (Yu-kai Chou): all 8 core drives are activated during Boss Fight; other phases use subsets. This was a deliberate choice to make the Boss the emotional peak.
Duolingo mechanics adapted: streak freezes (500 gems, max 2/week), hearts system (boss hints cost HP), endowed progress (bar starts at ~15%).
# Section 0: Pre-Session (<2 seconds, server-side, invisible)
Purpose: Everything that happens before the student sees anything. Assembles the entire session plan server-side so the experience feels instant.

What it organizes:
Load student profile (PISA levels, mastery map, transition skills, streak, trajectory)
Load assignment (textbook, chapter, objectives, deadline, mode)
Hard block: content pool must have ≥85 tagged items or session CANNOT proceed — this prevents half-baked sessions
Calculate difficulty band: floor/target/ceiling based on PISA level + teacher overrides
Assemble session plan: select specific items for each of the 7 phases, ensuring Bloom’s coverage and transition skill representation
Generate lesson brief (4-5 bullet summary from textbook — disappears after 30 sec, never shown again)
[HOOK: Endowed Progress] Progress bar starts at ~15% filled — student feels they’ve already started before doing anything
# Section 0-A: Theme Preview (NEW — Pre-Homework, student-paced)
Purpose: Optional pre-session warmup. Lets the student explore the topic before any pressure or scoring. No timers, no XP, no penalties — pure curiosity priming.

8 components (student can browse any/all):
Summary — short topic overview from textbook
Better Explanation — reworded for clarity, alternative angle
Examples — concrete worked examples
Real-Life Research — how the concept appears in the world
Personal Hook — why a student their age might care
Why It Matters — long-term relevance / academic ladder
Industry Application — careers and fields that use it
Additional Materials — videos, articles, supplementary links in any language
Choice rationale: Demos showed students felt dropped into content cold. Theme Preview gives the curious ones a self-directed on-ramp without forcing the rushers to slow down.
# Section 0-B: Flash Cards (NEW — Pre-Homework reference deck)
Purpose: All key formulas, concepts, and rules for the topic, one per swipeable card. Acts as a personal reference deck the student can return to during the homework session.

How it’s organized:
One concept/formula/rule per card — no walls of text
Swipeable interface, student-paced, no scoring
Returnable during homework phases (quick-access overlay) — NOT a hint, just a reference
Ends with explicit "Start my Homework" button — student decides when they’re ready
Choice rationale: Removes the "I forgot the formula" friction without giving away answers. The student earns the right to consult their own deck.
# Section 1: Phase 1 — ENGAGE + REMEMBER (Memory Sprint, flexible format)
Purpose: Pull the student in within the first 30 seconds AND warm up recall from prior chapters. Memory Sprint is no longer just "questions" — it’s a flexible fast-paced format pool.

How it’s organized:
Hook Game (30 sec): fast-tap classification, intentionally EASY and ADDICTIVE. 90%+ success rate expected. 50 XP per tap, speed bonus, variable "SUPER!" animation on ~20% of answers.
Streak Display + Lesson Brief flash (30 sec, disappears permanently). Streak tiers: 1-day → 7-day → 14-day → 30+ day with increasing rewards.
Streak Freeze: costs 500 earned gems (not real money), max 2/week.
Memory Sprint (UPDATED — flexible format): ≤2 minutes, fast-paced, quick-dopamine, prior chapters only. NOT just questions.

Approved Memory Sprint formats (any of, mixed within the 2 min):
MC / Binary — quick yes-no or 4-option taps
Speed Match — pair concepts to definitions
Flash Sprint — rapid flashcard recall
Fill-in-Blanks Race — type or tap missing word
Order the Steps — sequence a procedure correctly
Choice rationale: Demos showed pure question lists felt like a quiz, killing the dopamine. Mixing formats keeps the 2 min varied while still hitting prior-chapter recall.
# Section 2: Phase 2 — UNDERSTAND (Story Mode, REWRITTEN)
Purpose: Deliver new textbook content through a complete story arc, not disconnected segments. Demos showed previous Story Mode was bland and felt like a textbook read-aloud.

Story arc structure (mandatory):
Problem — a real situation that demands the concept
Struggle — character tries familiar approaches and they fail
Discovery — the concept is introduced as the missing tool
Solution — character applies it, the situation resolves

How it’s organized:
3 segments × 90 seconds each, with checkpoints between each segment.
Real-world examples preferred over invented scenarios — actual cases, real people, real places when possible.
IELTS-style reading comprehension on checkpoints (inference, main idea, vocabulary in context) — not just recall.
Stranger Test quality gate: any reader unfamiliar with the topic should still understand the story on first read. If they can’t, the segment fails QA.
CRITICAL interleaving: Seg1 → Game1 → Seg2 → Game2 → Seg3 → Game3. Interleaving improves retention (Roediger & Karpicke).
Textbook page reference (Darslik: sahifa X) pulses on checkpoint failure -- guides student back to source without giving away answer.
Skimming detection: <30 sec on a 90-sec segment → flagged.
CPA progression for math (Concrete → Pictorial → Abstract) built into story segments.
# Section 3: Phase 3 — APPLY (Game Breaks)
Purpose: Practice through game mechanics. Confirms the interleaving with Story Mode.

How it’s organized:
Game selection: 1 reinforcement game (practice what was just learned) + 1 stretch game (push slightly beyond) + 1 transition skill game (target PISA level-up skill).
Each game has [HOOK] markers for specific engagement mechanics (variable rewards, near-miss feedback, combo multipliers).
In-game adaptation: <60% → tier down, >90% → tier up, 70-80% = flow zone. This runs in real-time, not between games.
Choice rationale: 3 games with different purposes prevents repetition fatigue while ensuring both consolidation and growth.
## New Game: Notebook Capture (hand-written/drawn tasks)
Purpose: Force the student off the screen and back into a physical notebook for tasks that benefit from hand-writing or hand-drawing — long-form math derivations, free-body diagrams, geometric constructions, labelled biology diagrams, hand-drawn timelines, essay drafts, calligraphy. Student does the work on paper, takes a photo, uploads it. AI vision evaluates against rubric.

Why it exists: Cognitive science (Generation Effect, Embodied Cognition, Mueller & Oppenheimer pen-vs-laptop research) shows physical writing improves retention 25-40% over typing. NETS was previously screen-only — this closes that gap without breaking the digital game loop.

How it works:
Frequency: ~1 in every 3-4 sessions for Grades 5+; ~1 in 6 for Grades 1-4. NOT every session — overuse breaks the loop.
Where it fits: Phase 3 (replaces one game slot) OR Phase 4 (case requires hand-drawn diagram) OR Phase 6 (Mythical Boss almost always demands one).
Flow: prompt → work offline (timer frozen, app can be backgrounded) → tap UPLOAD → camera with live page-edge detection + focus + lighting validation → capture → confirm preview (auto-crop, face/PII auto-blur) → multi-page support (up to 3) → send.
AI Tier 3 multimodal vision: 10-second hard cap, OCR + diagram understanding, returns score + partial credit + specific feedback. Annotations overlaid on photo (green checks on correct parts, red circles on problems).
Privacy: encrypted, 30-day retention, teacher/parent only. Faces and personal identifiers blurred client-side BEFORE upload.
Anti-cheat: vision flags suspiciously perfect work (looks printed), identical submissions across students, handwriting baseline mismatch — routed to teacher review, NOT auto-failed.
Accessibility: visually impaired → audio recording task instead. Motor disability → teacher toggle off. No notebook → in-app sketch fallback capped at 80% XP (bypasses embodied cognition benefit).
Teacher controls: enable/disable, frequency 1-in-3 default (1-in-6 min, every-session max), pristine-notebook bonus toggle, dashboard review, manual score override.
## Interactive Game Catalog Integration
Catalog file: standards/NETS-Interactive-Game-Catalog.md (v1.0, 12 knowledge-gated games derived from classic board/card/puzzle mechanics).
Dual-catalog selection rule: Phase 3 picks 3 games per session. At LEAST 1 must come from the Interactive Catalog AND at LEAST 2 must come from the Default 16-mechanic pool (§6). The Interactive Catalog is a complement, never a replacement.
Games in the catalog:
Tic Tac Toe vs AI — minimax duel; correct answer places chosen tile, wrong = random.
Connect Four vs AI — 7x6 drop grid; column choice gated by question.
Codebreaker — Mastermind-style code crack; each guess attempt gated.
Memory Match Blitz — Concentration grid; matched pairs confirmed by question.
Reaction Chain — 6-10 node chain; 3 wrong in a row breaks the chain.
Word Ladder Climb — transform start word to target one letter at a time.
Puzzle Lock (Sliding Tile) — 3x3/4x4 sliding puzzle; wrong = random tile slides.
Blackjack 21 Knowledge Edition — Hit/Stand gated; wrong = AI makes the choice.
Territory Conquest — simplified Risk 1v1 area control; failed attack loses territory.
Escape Room: Subject Lock — 4 locked objects, cross-hinting clues, time limit.
Bridge Builder — physics bridge construction; wrong answers waste budget.
Minefield Navigator — hidden-mine grid; wrong = forced adjacent move (likely mine).
When to pick a Catalog game vs a Default game: Catalog games shine when the topic maps cleanly to a spatial/strategic metaphor (chronology → Puzzle Lock, geography → Territory Conquest, periodic table → Codebreaker), when engagement has dipped, or for Big Boss / Mythical Boss cross-subject reviews. Default mechanics win for high-density Remember/Understand drilling, narrative-heavy content, physical (notebook) work, Recovery Sessions, and below-PISA-L2 students who need lower cognitive overhead.
## New Game: Tic Tac Toe vs AI
Purpose: Turns recall practice into a strategic micro-duel. The student isn’t answering a quiz, they’re trying to beat an opponent — the question is just the gate.

How it works:
AI asks a question → correct answer = student places their tile where they choose; wrong answer = a random tile is placed for the student.
3 chances total per game (3 wrong = forfeit).
Adaptive difficulty: question difficulty decreases if the student is losing.
Minimum success rate enforced: 60% (otherwise dropped to easier question pool).
AI plays optimally (perfect tic-tac-toe), so the student MUST answer correctly to have any chance.
Choice rationale: Combines an externally familiar game with content recall. Beating an "unbeatable" AI by knowing the material is a strong intrinsic reward.
# Section 4: Phase 4 — ANALYZE (Real-Life Challenge, REWRITTEN)
Purpose: Transfer knowledge to real-world context with the student in the driver’s seat. Demos showed previous version was too detached — generic third-person scenarios. Now: first-person expert POV.

How it’s organized:
First-person Expert POV: "You are the doctor / engineer / historian / lawyer making this call." Student IS the decision-maker, not an observer.
Real cases preferred over invented scenarios — actual historical events, real legal cases, documented engineering decisions.
Tricky multiple-path questions with plausible wrong answers (distractors that look right to a novice) — forces real reasoning, not pattern matching.
Explanation required after answering: student must justify their decision. AI evaluates the reasoning, not just the choice.
Eligibility gate: PISA L2+ required. Below L2 gets adaptive quiz instead.
Uzbek cultural context preferred where applicable: Uzbek names, locations, som currency.
[HOOK: Reward Chest] on completion with variable XP + random avatar item.
# Section 5: Phase 5 — CONSOLIDATION
Purpose: Lock knowledge before Boss Fight. Creates psychological "I’m ready" moment.

How it’s organized:
Technique selected by grade band AND content type (not one-size-fits-all).
Memory Palace is default for Grades 3+. Younger students get simpler mnemonic games.
Choice: the consolidation phase exists specifically to reduce boss anxiety. Students who feel prepared perform 20-30% better (testing effect research).
# Section 6: Phase 6 — BOSS FIGHT (3-Tier AI Boss System, UPDATED)
Purpose: The emotional peak of the session. Previously a single boss per session — now a 3-tier system that scales from per-homework to mythical rare events.
## Tier 1: Sub Boss (every homework)
The standard per-homework boss — current topic only.
HP: 50 (Grades 1-4), 100 (Grades 5-8), 150 (Grades 9-11).
Damage scaled by PISA tier (Easy=L3, Med=L4, Hard=L5-6).
Combo system: 3 correct in a row → 2x damage.
Hint tax: hints heal the boss.
AI Tier 3 evaluates open-ended answers; accepts multiple valid interpretations for History/Reading.
## Tier 2: Big Boss (weekly)
Appears once per week, cross-subject.
Targets the student’s 3 weakest standards (pulled from mastery map).
Pays 2x XP across the fight.
Choice rationale: forces spaced retrieval across subjects, not just within today’s topic.
## Tier 3: Mythical Boss (random, <5% chance)
Random spawn at <5% probability per session — cannot be farmed.
PISA 5-6 difficulty only, ZERO hints allowed.
Massive rewards: 5x XP, exclusive title, avatar frame, Hall of Fame entry.
Choice rationale: variable-ratio reward at the rarest tier creates a long-tail dopamine hook (the slot machine effect, but earned through skill).
## Shared boss rules (all tiers):
Victory: Star calculation (1★/2★/3★) → animated reward chest → social comparison (opt-in).
Defeat: max 3 attempts → Socratic tutoring (NEVER reveals answer) → regenerated questions on retry → teacher notification if all 3 fail.
[HOOK: Near-Miss] "Boss HP: 15 qoldi! Ozgina qoldi edi!" creates urgency.
Open-reasoning only for Grade 5+ (no MCQ at boss level).
# Section 7: POST-BOSS — AI Analysis Engine (<10 sec)
Purpose: Invisible server-side analysis that drives all downstream reporting and adaptation. Student never sees this phase.

What it computes:
Performance analysis: by phase, by Bloom’s level, by standard, by transition skill.
Root cause identification: WHY the student failed, not just WHAT they got wrong. Choice: most LMS only track correctness; NETS tracks the reasoning gap.
PISA level recalculation with IRT weighting + exponential decay (recent performance weighted more).
Trajectory check: 3+ declining sessions → triggers Boost Mode automatically.
Generates 3 outputs: next session plan, teacher report (detailed), parent report (simple, actionable, in parent’s language, no PISA jargon).
# Section 8: REMEDIATION + REFLECTION
Purpose: Turn failures into bonus XP opportunities (reframing) and create hooks for next session.

How it’s organized:
Weakness display: max 2-3 areas shown, framed as "bonus XP opportunity" not "you failed at". Choice: growth mindset framing (Dweck) reduces learned helplessness.
Micro-exercises: simpler scaffolded versions targeting specific failure points.
Spaced repetition queue update (SM-2 algorithm) — failed items get shorter intervals.
[HOOK: Reward Chest] final session chest with variable rewards.
[HOOK: Zeigarnik Effect] next lesson teaser creates incomplete mental loop — student thinks about next session involuntarily.
Chapter progress ring creates visual tension to fill (endowed progress + completion drive).
# The NETS Loop (Duolingo-adapted)
TRIGGER → ACTION → VARIABLE REWARD → INVESTMENT → (next day: internal trigger)

# Cross-Cutting Systems (UPDATED)
## Mystery Box (REWORKED — case-opening event, NOT gambling)
Purpose: Variable-reward chest the student EARNS through performance. Removed any random-pay mechanics; appears either randomly during sessions or pre-placed by the engine after milestones.

Reward pool (performance-weighted draw):
XP Boost — 40%
Extra AI (bonus AI tutor session) — 25%
Avatar Frame — 15%
Customization (color/theme) — 10%
Coupon — 7%
Mythical Boss Ticket — 3%
Choice rationale: Case-opening framing (the box is THERE, the student opens it) avoids slot-machine ethics while keeping the variable-ratio dopamine hit. Mythical Boss Ticket at 3% creates the lottery dream without being predatory.
## "Did You Know..." Loading Insights (NEW)
Purpose: Use the unavoidable phase-transition loading moments for tiny knowledge insertions. Free attention real-estate.
1-2 sentences max, theme-relevant to the current topic.
1 insight per phase transition (so 6-7 per session).
No quiz follow-up — pure curiosity injection, no pressure.
Choice rationale: turns dead time into micro-learning without adding session length.
## Answer Checking — Dual Pathway (UPDATED)
Purpose: Two routes for evaluation depending on question type. Eliminates the previous "AI checks everything" latency and cost problem.

Pathway A — Hardcoded (instant):
Used for: multiple choice, matching, numeric answers.
Rule-based exact match against the answer key.
Zero AI cost, zero latency.
Pathway B — Open Reasoning (AI-evaluated):
Used for: typed open responses, justifications, Real-Life Challenge explanations, boss reasoning.
AI Tier 3 evaluates against a per-question rubric in real time.
5-second timeout fallback: if AI doesn’t respond, falls back to keyword matching against the rubric.
Choice rationale: keeps the experience fast for 80% of questions and reserves AI cost for the questions that genuinely need judgment.
## Assessment Overhaul (NEW grading tiers)
Purpose: Replace the old binary pass/fail with a tiered emotional response system that matches what the student actually achieved.


Choice rationale: The old "you passed" message felt the same at 60% and 100%. Tiered responses give the student something to chase. Duolingo Mode for <60% prevents the framework from rubber-stamping failure.

## The Engagement Philosophy
We use White Hat gamification (epic meaning, accomplishment, empowerment) as primary drivers and Black Hat gamification (scarcity, loss avoidance, unpredictability) sparingly and transparently. Per NETS Principle 3: we celebrate LEARNING progress, not compliance. Zero pay-to-win. Zero real-money purchases.
Frameworks integrated:
Hook Model (Nir Eyal): Trigger → Action → Variable Reward → Investment — one full cycle per session
Octalysis (Yu-kai Chou): 8 Core Drives mapped to specific phases
Compulsion Loop: Anticipation → Action → Reward — dopamine hits at variable intervals
Duolingo Mechanics: Streaks, XP leaderboards, hearts system (as hint tax), streak freeze, leagues, reward chests, legendary challenges — adapted for education

## 0.3 Difficulty Band Calculation
Inputs:
Student's current PISA level in this subject
Grade's target PISA level
Teacher overrides (if any — SPEC §14)
Student's weakest domain areas
Algorithm:
difficulty_band = {
floor: max(student_pisa - 1, 0),              // Never below PISA 0
target: student_pisa,                          // Most items at current level
ceiling: min(student_pisa + 1, 6),             // Stretch items one level above
teacher_override: teacher_floor || teacher_ceiling || null
}
Output determines: Which items from the pool get selected, how hard the boss questions will be, which game mechanics are appropriate, which transition skills to target.
## 0.4 Session Assembly
[AI: Tier 1] selects specific content items from the pre-generated pool for each phase:
session_plan = {
phase_1_memory_sprint: SELECT 5-8 recall_items
FROM previous_chapters
WHERE spaced_repetition_due = true
ORDER BY priority DESC                       // Wrong > approaching_threshold > stale

phase_2_story_mode: SELECT 3 narrative_segments
FROM this_chapter
WHERE standard_ref IN assignment.learning_objectives

phase_3_game_breaks: SELECT 3 game_items
WHERE at_least_1_reinforcement                // At student's current level
AND at_least_1_stretch                        // One level above
AND at_least_1_transition_skill_match          // Targets in-progress skill
AND no_mechanic_repeats_more_than_2x

phase_4_real_life: SELECT 1 transfer_scenario
WHERE pisa_level >= 2                         // Below L2 → extra adaptive quiz instead
AND cultural_context = "uzbek"

phase_5_consolidation: SELECT 1 mnemonic_exercise
WHERE technique matches grade_band

phase_6_boss: SELECT 5-7 boss_questions
WHERE 40% easy + 40% medium + 20% hard        // SPEC §5.6 line 691
AND covers_at_least_2_transition_skills

phase_7_reflection: SELECT 1 reflection_prompt
// Will be dynamically adjusted based on performance
}
Transition skill coverage verification: At least 2 different transition skills must be targeted across the session. At least 1 must match the student's in_progress skills. If this fails → re-select items until constraint is met.

# SECTION 0-A: THEME PREVIEW (NEW)
## Position and Purpose
Position: First thing the student sees when opening a homework assignment, after the invisible Pre-Session load (Section 0).
Purpose: Hook the student. Connect the topic to their world. Explain why it matters. Equip the mental model BEFORE any practice begins. This is the bridge between reading the textbook and doing the gamified homework — the place where engagement was previously dying.
Duration: 2-3 minutes, student-paced. No timer. No pressure. No XP.
## The 8 Required Components
1. Summary of Book Content — a cleaner reframing of the chapter, plain language, better visual hierarchy. Not a rewrite, a refocusing.
2. Better Explanation — concepts the textbook assumes the student already understands, clarified. Fills the gap between textbook language and student comprehension.
3. Examples — additional worked examples beyond the textbook, step-by-step.
4. Real-Life Research — the origin story of the concept. Who discovered it? What problem did it solve? What changed in the world?
5. Personal Hook — first-person address connecting topic to the student's own life. "Remember when you saw...?" / "Maybe you've wondered why...?"
6. Why This Matters — explicit statement of real-world relevance, not "it's on the exam".
7. Industry Application — how the concept is used in real industries, jobs, and technology.
8. Additional Materials — curated external resources (videos, articles, simulators). Language does NOT matter. English, Russian, or Uzbek are all permitted. Student picks.
## Design Rules
No quiz, no pressure. Exploration mode. There are no answers to give, so there are no wrong answers.
Student-paced. No timer. Student scrolls/clicks at their own speed and may skip components.
Visual-first. Diagrams, photos, short video clips preferred over walls of text.
First-person POV throughout. "You" not "the student". Direct address only, never third-person.
Ends with explicit transition to Flash Cards: "Before we start — here are the key ideas you'll need."
# SECTION 0-B: FLASH CARDS (NEW)
## Position and Purpose
Position: Between Theme Preview (Section 0-A) and the 7-phase Homework Engine.
Purpose: Surface every formula, concept, rule, and definition the student will need during the homework. The last reference moment before the challenge begins. Doubles as a quick-access reference deck DURING the homework.
Duration: 1-2 minutes for the initial pass. Available throughout the homework on demand.
## What the Card Pool Must Contain
All Key Formulas — every formula, equation, or rule the student will encounter in the homework. One per card.
All Key Concepts — definitions, terminology, classifications. "What is an oxide?" / "What does valence mean?"
All Key Rules — conditions, exceptions, boundaries. "Combustion requires 3 things: fuel, oxygen, heat. If any one is missing, fire stops."
## Design Rules
One concept per card. No cramming.
Swipeable / scrollable. Student flips through at their own pace.
No testing. Reference, not assessment. No right/wrong.
Returnable during homework. A "Flash Cards" button is available throughout the session as a quick-access overlay. NOT a hint (no penalty), just a reference.
Ends with the explicit "Start my Homework" button. Only after this press does the session timer start and XP accrue.

## 1.1 The Hook Game (30 seconds)
[HOOK: Action — Low Friction] A fast-tap game that is intentionally EASY and ADDICTIVE. XP and streaks start accumulating immediately.
What the student sees: A rapid-fire classification game. Items appear on screen, student taps the correct category. No typing. No thinking. Pure reflex + recall.
Subject-specific hooks:

Psychological principle: Csikszentmihalyi's flow state — start with LOW challenge / HIGH reward to pull the student in, then gradually increase challenge. The hook game is NOT assessment. It's momentum.
AI selects game type based on subject + student's weakest domain. Sets initial difficulty LOW — 90%+ success rate expected.
What AI tracks: Response speed, accuracy, which categories cause hesitation — this data feeds ALL later phases.
Scoring:
50 XP per correct tap
Speed bonus: +10 XP if answered in <2 seconds
[HOOK: Variable Reward] Random "SUPER!" animation on ~20% of correct answers → +25 bonus XP (variable ratio reinforcement — Skinner)
10-second timer, 8–12 items

## 1.3 Flashcard Sprint (Remember Gate)
Purpose: Test recall of PREVIOUS chapters. Not current topic. This is spaced repetition in action.
Parameters:

Item Selection Algorithm (SM-2 Modified):
1. GET all recall_items from chapters BEFORE current chapter
2. PRIORITIZE by forgetting curve:
Priority 1: Items student got WRONG in previous sessions
Priority 2: Items approaching forgetting threshold (Ebbinghaus: retention < 60%)
Priority 3: Items answered correctly but not seen in 7+ days
Priority 4: Items answered correctly, seen recently (filler if needed)
3. FILTER to target count, balanced across prior chapters
4. RANDOMIZE order within priority bands
Scoring:
Correct: +100 XP
Streak bonus: +10 XP × current_streak_within_sprint (resets per sprint)
Speed bonus: +10 XP if answered in <5 seconds
[HOOK: Variable Reward] "MUKAMMAL!" animation on 3+ streak → bonus XP varies (50–150, randomly)
No penalty for wrong answers — this is warm-up, not assessment
Adaptation Gate:
IF accuracy >= 80%  → Student ready for new content. Proceed to Phase 2.
IF accuracy 60–79%  → Proceed but FLAG for teacher review. AI notes which standards decayed.
IF accuracy < 60%   → ROUTE TO REMEDIATION before new content begins.
System inserts a 2-minute micro-review targeting the failed standards.
Then proceeds to Phase 2 with adjusted difficulty band (floor lowered by 0.5).
Standards reference: Every recall item displays its standard code in the post-session review: UZ-MATH-5-NATNUM-03 — Natural sonlarni yaxlitlash
Research basis: Retrieval Practice (Roediger & Karpicke, 2006) — active recall is 3x more effective than re-reading. Forgetting Curve (Ebbinghaus, 1885) — without spaced review, 70% of learning is lost within 24 hours.

## 1.4 Memory Sprint — Format Flexibility (UPDATED 2026-04-07)
The original spec said "single-answer recall" — too narrow. All four AI demos produced typed quiz lists. The new rule: Memory Sprint can be ANY fast-paced format that hits prior chapters in ≤2 minutes and delivers quick dopamine.
Approved formats:
• Quick MC / Binary — 5-8 tap-to-answer questions
• Speed Match — drag-and-drop term ↔ definition / formula ↔ result
• Flash Sprint — rapid-fire flashcards, swipe Know / Review
• Fill-in-Blanks Race — complete sentences with missing keywords, timed
• Order the Steps — arrange procedure steps in correct order
Content creator chooses the format that fits the subject. Mixing formats within a Sprint is allowed. All formats must have a Start button, instant per-item feedback, and a final score display.

## 2.3 Narrative Cultural Framework
[HOOK: Epic Meaning — Octalysis Core Drive 1] The student is the HERO of the story. They're not passively reading — they're discovering, exploring, solving.

Character names: Always Uzbek — Sardor, Malika, Alisher, Nargiza, Jasur, Dilnoza

## 2.4 Checkpoint Questions
Purpose: Comprehension gates, NOT assessment. They check "did you understand what you just read?"
Rules:
1–2 questions between each segment
Student MUST answer correctly to proceed to the next segment
On wrong answer: show explanation referencing the EXACT textbook page, then re-ask
Maximum 2 retries per checkpoint. After 2 failures: [AI: Tier 2] provides a simplified version of the question with scaffolding
Format: varies by grade (MC for Grades 1–2, type answer for 3+)
Skimming detection: AI tracks time per segment. If student spends <30 seconds on a 90-second segment → flagged as "likely skimmed." This flag increases the checkpoint difficulty slightly and adds an extra Story Mode micro-segment at end if pattern persists.
## 2.5 Book Reference Integration
Every segment shows a persistent 📖 Darslik: sahifa X (Textbook: page X) reference in the corner.
On checkpoint failure: The textbook page reference PULSES and enlarges — a visual nudge to actually read the book. This reinforces Criterion 1: the book is the lesson, NETS is the transformer.
Research basis: Narrative Retention (Willingham, 2004) — stories are retained 22x better than raw facts. CPA approach (Leong et al., 2015) — structured concrete-pictorial-abstract progression for mathematical understanding.

## 2.6 Story Mode — Story Arc Mandatory (UPDATED 2026-04-07)
All four demos produced textbook paragraphs and called them narratives. The new rule: every Story Mode segment must follow a complete story arc — Problem → Struggle → Discovery → Solution.
Problem — opens with a real problem or mystery. Not "Today we'll learn about..." but "In 1774, a priest named Priestley heated a red powder and something impossible happened..."
Struggle — character tries familiar approaches and they fail. Tension builds.
Discovery — the new concept arrives as the missing tool. Textbook content lands HERE.
Solution — character applies the concept, situation resolves. Student gets the answer but EARNED it by following the logic.
Real-world examples preferred. If the concept has a real historical discovery or documented case, use it as the spine. Invented scenarios are a fallback.
IELTS-style comprehension on checkpoints. Test main idea, inference, comparing across segments, vocabulary in context — NOT fact recall. Fact recall belongs in Memory Sprint.
### The Stranger Test (Quality Gate)
Before deployment, every Story Mode segment must pass: if a person who has never seen this textbook content can follow the segment and answer the checkpoint question using only the narrative logic (not prior knowledge), it passes. If it requires the reader to already know the content, it fails. Failing segments return to the content pipeline for rewrite.

### Game: Tile Match
What student sees: Grid of 8–12 face-down tiles (4–6 pairs). Tap to reveal, find matches.
Gameplay:
Tap first tile → reveals content (term, image, formula, or concept)
Tap second tile → reveals content

### Game: Adaptive Quiz (IRT-Calibrated)
What student sees: Standard quiz format — question, answer input, feedback. But difficulty adapts after each answer.
IRT Implementation:
Each question has pre-calibrated parameters:
- difficulty (theta): -3.0 to +3.0
- discrimination (alpha): 0.5 to 2.0
- guessing (c): 0.1 to 0.25

Algorithm:
1. Start at theta = 0 (grade-level expectation)
2. Present question with difficulty ≈ current theta
3. IF correct: theta += 0.3
IF incorrect: theta -= 0.3
4. Select next question with difficulty ≈ new theta
5. Repeat for 5–7 questions
6. Final theta = student's ability estimate for this standard

PISA mapping:
theta -3 to -1  → PISA Level 1
theta -1 to 0   → PISA Level 2
theta 0 to 1    → PISA Level 3
theta 1 to 2    → PISA Level 4
theta 2 to 3    → PISA Level 5+
XP: Based on final theta achieved (higher theta = more XP)
Timer: 2–3 minutes total

### Game: Notebook Capture (NEW 2026-04-07)
A task type that forces the student off the screen and back into a physical notebook for work that genuinely benefits from hand-writing or hand-drawing — long-form math derivations, free-body diagrams, geometric constructions, labelled biology diagrams, hand-drawn timelines, essay drafts. The student does the work on paper, takes a photo, uploads it, and AI vision evaluates the result against the rubric.
Why it exists: cognitive science (Generation Effect, Embodied Cognition, Mueller & Oppenheimer's pen-vs-laptop research) shows physical writing improves retention 25-40% over typing. NETS was previously screen-only — Notebook Capture closes that gap without abandoning the digital game loop.
When it appears: roughly 1 in every 3-4 sessions for Grades 5+. Lower grades (1-4) get them less often (1 in 6) because younger students need more digital scaffolding. NOT mandatory in every session — overuse breaks the game loop.
Where it fits: Phase 3 (Game Breaks) as a 5-min deep dive replacing one game slot, OR Phase 4 (Real-Life Challenge) when the case requires a hand-drawn diagram, OR Phase 6 (Boss Fight) for higher grades — Mythical Boss almost always requires it.
### Notebook Capture — Task Flow
1. Task card displays the prompt: "📓 Notebook Task — Sketch the free-body diagram for a block sliding down a 30° ramp. Label all forces. Show the net force vector."
2. Instruction: "Open your notebook. Do the work. When you're done, tap UPLOAD to take a photo."
3. Student works offline — no timer pressure beyond a soft 10-min suggested cap. App can be backgrounded.
4. Student taps UPLOAD: opens camera (or photo picker as fallback).
5. Pre-upload validation runs client-side: focus check (Laplacian variance > 100), page detection (rectangular bound found), lighting check (mean brightness 60-200). If any fails: prompt to retake with a specific tip.
6. Image uploads to AI Tier 3 vision model. OCR + diagram understanding extracts the student's work, compared against the rubric.
7. Within 8 seconds, AI returns: score 0-100%, partial credit breakdown, specific feedback ("Your free-body diagram correctly shows gravity and normal force, but the friction vector is pointing the wrong way. Friction opposes motion — re-check the direction.")
8. Feedback shown to student WITH the photo and AI annotations overlaid (small red circles around problem areas, green checks on correct parts).
9. Student can ACCEPT the score and move on, or RETRY with a corrected page if score < 60%.
### Notebook Capture — Privacy & Anti-Cheat
Photos are stored encrypted, retained 30 days for re-evaluation/audit, then deleted. Teacher and parent can view; nobody else.
If the AI vision detects faces or visible personal identifiers (name on the page header), those regions are blurred client-side BEFORE upload.
Cheating detection: vision model flags suspiciously perfect work (looks printed/copied), identical submissions across students, or work that doesn't match the student's prior handwriting baseline. Flagged work goes to teacher review, NOT auto-failed.
Accessibility: visually impaired students get a verbal description audio task instead. Motor disability: teachers can toggle the mechanic OFF per student. No notebook? In-app sketch tool fallback (stylus or finger drawing) — capped at 80% XP because it bypasses the embodied cognition benefit.
### Notebook Capture — Teacher Controls
Teachers can disable the mechanic for their class (default ON for Grades 5+), adjust frequency (1-in-3 default, 1-in-6 minimum, every-session maximum), toggle the "pristine notebook" presentation bonus, review all submitted photos via the teacher dashboard, and manually override AI scores.

### When to pick a Catalog game vs a Default game
Pick a CATALOG game when: the topic maps cleanly to a spatial/strategic metaphor (chronology → Puzzle Lock, metabolic pathways → Reaction Chain, periodic table → Codebreaker, geography → Territory Conquest); engagement has dipped over the last 3 sessions; the session is a Big Boss or Mythical Boss cross-subject review (Territory Conquest, Escape Room, or Bridge Builder are configured for this); or the student has already seen the same Default mechanic twice this week.
Pick a DEFAULT game when: the Bloom's target is Remember/Understand and needs maximum repetition density (Spaced Flashcards, Memory Sprint, Tile Match); content is narrative-heavy (Story Mode, Why Chain); the task requires physical hand-work (Notebook Capture); the student is in a Recovery Session or below PISA L2 (Default mechanics have lower cognitive overhead); or the Interactive Catalog subject-compatibility matrix marks the Catalog option as weak/incompatible for the current subject.
Device / context constraints: Bridge Builder and Minefield Navigator carry higher render cost and should be skipped on low-end devices; Blackjack 21 is math-only; Word Ladder Climb is language/biology only. ## 4.6 Real-Life Challenge — First-Person Expert POV (UPDATED 2026-04-07)
All demos used third-person scenarios ("Tomir's plant is dying", "Salimjon is in his garage"). Students were observers, not decision-makers. Every Real-Life Challenge must now put the student in the role of THE EXPERT who must solve a real case.

Unchanged from the existing Phase 6 specification. Tests current lesson content at the student's PISA-calibrated difficulty. 3 attempts max. Socratic tutoring on failure. Regenerated questions on retry.
### Big Boss (weekly, replaces that week's final Sub Boss)
Auto-assigned once per week per student. Cross-subject scope: AI identifies the student's 3 weakest standards across ALL completed subjects and builds the question set around them.
Difficulty: one tier above the student's current PISA level in each subject. If the student is Math 1.7, Big Boss Math questions target PISA L2-3.
Format: same as Sub Boss — open reasoning, no MC for Grade 5+, Socratic hints available (cost HP).
Reward: 2× XP. Weekly leaderboard badge. Streak freeze token if score > 80%.
Failure: weak standards persist into next week's Big Boss until mastered.
### Mythical Boss (random elite event, <5% chance)
Trigger: random appearance, less than 5% chance per homework session. The student cannot predict, force, or farm it. Appears as a surprise: "⚠️ A Mythical Boss has appeared!"
Scope: ALL subjects passed up to current point. Questions span Math, Science, Biology, History, anything completed.
Question type — one of: (1) Real-World Case Scenario requiring multi-subject reasoning, or (2) Advanced Mathematical Challenge with high-level equations or cross-disciplinary math.
Difficulty: PISA L5-6 regardless of student level. Designed to overwhelm. Most students will fail — by design.
Hints: ZERO. No Socratic tutoring. ONE attempt only.
Reward if passed: 5× normal session XP, exclusive title ("Mythical Slayer", "The Chosen One"), permanent exclusive avatar frame, entry into Mythical Hall of Fame (opt-in, class-visible).
If failed: NO penalty. Small participation XP. "The Mythical Boss retreats... for now."
Psychological role: aspiration, social buzz, legend status. The rarity makes it legendary.

# SECTION 8-A: DUOLINGO MODE (NEW 2026-04-07)
Triggered when a student scores below 60% on a homework session. The student MUST redo the failed work until they reach 60%. No skip. No XP for the failed attempts.
## Step-by-Step Flow
1. Immediate feedback: "You scored X%. You need 60% to pass. Let's work on what you missed."
2. Task-by-task review: the system identifies which specific questions the student got wrong and re-presents them with additional scaffolding (hints, simplified versions).
3. Adjusted difficulty: re-attempt questions are one difficulty tier lower.
4. Repeat until 60%: no upper limit on attempts, but the teacher is auto-flagged after 3 failed sessions on the same topic.
5. No XP penalty: XP is only awarded once the student crosses 60%. Previous failed attempts earn 0 XP (not negative).
## Why It Exists
The original framework had no clear pass/fail threshold and no remediation loop. A student who failed simply moved on. Duolingo Mode closes the loop by refusing to rubber-stamp failure while keeping the framing growth-mindset positive.
# SECTION 8-B: ASSESSMENT CELEBRATION TIERS (NEW 2026-04-07)
Replaces the old binary pass/fail with a tiered emotional response system that matches what the student actually achieved. Every score range has a distinct UI response, reward calculation, and (where applicable) follow-up offer.
## Score-Tier Matrix
100% — PERFECT. On-screen celebration (confetti, animation, sound). Maximum XP. Temporary title: "Perfectionist".
99% — Almost Perfect. Tease: "So close! One more point!" Maximum XP minus 1%.
90-98% — Excellent. Strong praise. Full XP. Streak fire animation.
80-89% — Very Good. Bonus XP for hitting 80% on first attempt. Standard praise.
60-79% — Passing. Standard XP. Repass option offered: "You passed! Want to try again for a better score?"
Below 60% — Failed. Enters Duolingo Mode (Section 8-A). No XP for failed tasks.
## Repass Option (60-79%)
If accepted: homework restarts with regenerated questions (same standards, different numbers/scenarios).
If declined: session completes with current score and XP.
Maximum 2 repass attempts per homework.
## Did You Know — Loading Screen Insights (NEW)
Between phase transitions, the system displays 1-2 sentence "Did You Know..." insights related to the current lesson theme. 1 per phase transition. 5-second readable. Conversational tone. No quiz follow-up. Random off-topic facts are rejected. Example: "Did you know? The word 'oxygen' comes from Greek words meaning 'acid producer' — because scientists originally thought ALL acids contained oxygen. They were wrong."
## Answer Checking — Dual Pathway (NEW)
Hardcoded questions (MC, matching, numeric, formula input) are checked by rule-based exact match — no AI, instant feedback, <0.1s.
Open-reasoning questions (typed responses, justifications, essays, case analyses) are checked by AI Tier 3 in real time. AI receives: question, rubric, student answer, PISA level. AI returns: score 0-100%, partial credit breakdown, one-sentence feedback, all within 3 seconds. Hard 5-second timeout falls back to keyword matching against rubric concepts. AI never reveals the correct answer during evaluation.

# NETS Homework Engine — Unified Specification Summary
Source: standards/NETS-Homework-Engine-UNIFIED.md (v2.0, 2,072 lines, 20 sections + 2 appendices)
Status: Single Source of Truth — supersedes all previous specs
Target: 8.8M K-11 students, Uzbekistan
PISA 2022 Baseline: Math 364, Reading 336, Science 355, Creative Thinking 14/60
Goal: 80%+ below-L2 students reach L2-3 within 2 years
# Core Principles (§1)
Textbook is Source of Truth — NETS wraps content in engagement, never alters facts
Flow State is Mandatory — 70-80% success rate target (Csikszentmihalyi), adjust every 3-5 questions
Intrinsic Motivation > Extrinsic — Zero pay-to-win, all rewards celebrate learning
No Busywork Rule — Every task must scaffold ONE specific PISA transition skill
# Content Architecture (§2)
Hierarchy: Textbook > Chapter > Topic > Learning Objective > Content Block > Assessment Item
Learning Units: 1-3 per chapter section, each maps to one curriculum standard
Each LU generates 10-15 tasks across Bloom’s (4R, 3U, 3Ap, 2An, 1-2 E/C)
Dual Standard Codes: Primary descriptive UZ-MATH-5-FRAC-01 + alias dotted MAT.5.3.4.1
Mandatory tagging: textbook_ref, standard_ref, pisa_ref (with transition_skill), blooms_level, game_mechanic, prerequisites
# PISA Framework (§3)
7 levels (Below 1 through 6) defined for Math, Reading, Science
Continuous scale tracking per student (e.g., 1.7) via IRT-weighted performance
Domain breakdowns — Math: quantity/space/change/uncertainty; Reading: access/integrate/reflect; Science: identify/explain/interpret/evidence
History uses dual-domain mapping (Reading + Creative Thinking)
# Session Engine (§4)
Standard: 20-30 min (homework) | Extended: 40-50 min (in-class) | Recovery: 15-20 min
Initialization: load profile → load assignment → calculate difficulty band → select content → verify transition skill coverage → build session plan
# 7 Phases (§5)

Boss HP: Grades 1-4: 50 | Grades 5-8: 100 | Grades 9-11: 150
Boss Stars: 1★ = defeated | 2★ = ≤2 attempts, >50% HP | 3★ = 1st attempt, no hints, >80% HP
Boss Failure: Socratic tutoring → regenerated questions → max 3 attempts
# 16 Game Mechanics (§6, updated 2026-04-07)
Memory Sprint, Spaced Flashcards, Tile Match, Sentence Fill, Memory Palace, Story Mode, Adaptive Quiz (IRT), Mystery Box (case-opening), Movement Breaks (Grades 1-4 required), Why Chain, Peer Teaching (Grades 5+), Real-Life Challenge, Reflection Journal, Final Boss (3-tier: Sub/Big/Mythical), Tic Tac Toe vs AI (NEW), Notebook Capture (NEW — hand-written/drawn tasks evaluated by AI vision)
Plus Creative Lab (optional, Grades 5+, PISA L3+)
Complementary pool: the Interactive Knowledge-Gated Game Catalog (standards/NETS-Interactive-Game-Catalog.md, 12 games) is a sibling pool that Phase 3 draws from per the dual-catalog rule — ≥1 game from the Interactive Catalog and ≥2 from the Default 16 every session. See §6.9.
# Difficulty Adaptation (§9)
Real-time: <60% → tier down, >90% → tier up, 70-80% → flow zone
Secondary: 3 consecutive correct → +0.5 PISA sub-level, 2 consecutive wrong → -0.5
Emergency: <40% in game → insert Story Mode micro-segment for that standard
Cross-session: update PISA profile, spaced repetition, mastery map, trajectory
# Content Pipeline (§10)
4 phases: Ingestion (OCR) → Task Generation (3 AI tiers) → QA → Session Assembly
Minimum per topic: ~85 content items, all tagged/reviewed/traceable
Cost: ~$61 per textbook, ~$4,880 for Grades 5-8 pilot
# Gamification Economy (§11)
XP: 50-1000 per activity, streak multipliers (up to 2x at 30+ days)
Badges: Mentor, Streak Master, PISA Champion, Boss Slayer, etc.
Leaderboard: class-level only, weekly reset, anti-competition design
# Edge Cases (§12)
Missed homework: 3-day grace, condensed recovery session
Extended absence (7+ days): Catch-Up Mode with reduced boss HP
Boost Mode: triggered by 3+ declining sessions
Low engagement: reduce session length, increase game ratio
# Anti-Cheat (§13)
Question regeneration, response time analysis, pattern analysis, Socratic verification, device monitoring, copy-paste detection
Escalation: soft warning → parent → teacher → admin
# Teacher Controls (§14)
Session mode, grace period, boss HP modifier, hint toggle, difficulty floor, game selection, recovery limits
# Integrations (§15)
Kundalik/eMaktab: rosters, attendance, grades
Parent portal: PISA progression, recommendations, weekly summary
Admin analytics: per-school/region/national PISA levels
# JSON Schema (§16)
Task ID format: HW-{SUBJECT}{GRADE}-{TOPIC}-{SEQ}
Required fields: task_id, textbook_ref, curriculum_standard, pisa_level (with transition_skill), blooms_level, game_mechanic, content
AI tiers: 1=template, 2=mid-range (Haiku), 3=premium (Sonnet/Opus)
# Key Design Decisions (Appendix B)
Boss HP Grades 5-8: 80→100 (fixed math)
Final Boss: Level 3-6 tiered (resolved contradiction)
Session: 20-30 min default (50 min = Extended)
Dual standard codes (descriptive primary)
transition_skill field mandatory on every task

## 2.4 Transition Animation: "Whoosh + Letter Assembly"
Every swipe triggers a two-stage animation:
Stage 1 — WHOOSH (400ms)
Current panel slides out in the swipe direction
Motion blur on the outgoing panel (CSS filter: blur(4px) ramping from 0 → 4 → 0)
Subtle "whoosh" sound effect (can be muted)
Parallax: background image shifts at 60% of panel speed (creates depth)
Stage 2 — LETTER ASSEMBLY (600ms, overlaps with stage 1 by 150ms)
New panel's hero visual fades in at 40% opacity, scales up from 0.95 → 1.0
New panel's heading (e.g., "BETTER EXPLANATION") assembles letter-by-letter:
Letters appear in random positions scattered around final location
Each letter flies toward its final slot on a bezier curve, ~30ms stagger between letters
Letters arrive with a tiny overshoot (spring damping) → settle
Body text fades in as a block, 200ms after heading completes
Icon rotates/scales in separately (independent timing, ~800ms total)
Sound: Soft paper-rustle on whoosh, subtle typing-click bursts on letter assembly. All muteable.

## 3.2 Rotation Animation
Swipe left (advance) → carousel rotates counterclockwise:
Center card shrinks and slides left (1.0× → 0.7× scale, 0 → -120px X, 100% → 60% opacity, 0 → 4px blur)
Right card grows and slides to center (0.7× → 1.0× scale, +120px → 0 X, 60% → 100% opacity, 4 → 0 blur)
Hidden next-next card appears on the right at 0.7×
Cubic-bezier (0.25, 0.46, 0.45, 0.94) — "ease-out" feel, 500ms duration
All three cards move simultaneously, synchronized
Swipe right (go back) → same animation mirrored.
The circle nature means swiping past the last card loops to the first. A subtle "ding" sound + a brief pulsing ring around the counter indicates the loop.

## 3.5 Flip Animation
Trigger: Tap anywhere on the center card (only center is tappable)
Animation: Card rotates 180° around its vertical Y-axis
Timing: 500ms, easing cubic-bezier(0.4, 0.0, 0.2, 1) (material design "standard")

Background: full-bleed hero image for each story segment (problem / struggle / discovery / solution beats)

Checkpoint transition: background desaturates to 50%, a question card slides up from bottom with a soft pop

Wrong checkpoint: card shakes, textbook page reference pulses in the top-right ("📖 Darslik: sahifa 52") — pulsing animation draws the eye
## 4.4 Game Breaks (Phase 3) — Per-Game UI
Each mini-game has its own visual language. Common elements:
Game intro card (800ms): game name + icon + brief rule reminder, then slides away
Score HUD in top bar: current game score, best score, XP earned this game
Combo meter on the right edge: vertical bar that fills with each correct answer, empties on wrong
End-of-game summary: confetti burst (magnitude proportional to score), star rating (1-3), XP breakdown animation (each XP category flies in one at a time)
Tic Tac Toe vs AI specific:
3×3 board center-screen, cells glow softly when hoverable
Student taps a cell → cell highlights in blue → AI question modal slides in from the right

On wrong: X glitches to a random cell with distortion shader (400ms)
AI's turn: O etches into its chosen cell with a digital circuit trace animation (350ms)
Win: winning line draws across the board in gold, board explodes into particles
Mystery Box specific:
Appears as a floating 3D box center-screen, slowly rotating
Ambient glow in a random color (hints at rarity but doesn't spoil)
Student taps → box pauses → question modal appears
On answer: box CRACKS open in half (top half flies upward), contents shoot out like a CS:GO case-opening:
XP number flies out first
Then the reward item (if earned) with a flash burst
Rarity indicated by item glow: white (common) → blue → purple → gold → red (Mythical Boss Ticket)

Scenario introduction: slides in as a "case file" document — manila folder opens, papers inside flutter, revealing the scenario text

## 4.7 Boss Fight (Phase 6) — Epic Cinematic
Sub Boss:
Boss appears as a stylized illustrated character or abstract entity (subject-themed)

Victory = boss shatters into particles, starburst overlay, star rating animation (stars fly in from offscreen and slam into position)
Big Boss:
Same as Sub Boss but boss is larger (1.5×), has a different color palette (purple/gold), and the background is more dramatic (storm clouds, darker)

Mythical Boss:
Full cinematic cutscene intro (2-3 seconds):
Screen shakes
Dark vignette closes in
Red alert border flashes
Text materializes: "⚠ A MYTHICAL BOSS HAS APPEARED ⚠" with subject-appropriate mythological styling
Boss is massive (2.5× normal), glowing, with particle effects around it
Background: completely different — star field, abyss, or subject-specific legendary setting
Music shifts to an epic orchestral track (if enabled)
No hint button visible (it's disabled at all — enforce visually)
Victory animation is extreme: slow-motion explosion, camera zoom, screen flash, title card "MYTHICAL BOSS DEFEATED" with the student's exclusive title unveiling

Export format: Lottie JSON for complex animations (letter assembly, boss shatter, confetti). Native CSS/JS for simple transitions

Test matrix: iPhone SE (small screen), Galaxy A-series (mid Android), iPad (tablet), low-end school laptop (Chromebook). If any of these drops below 30 FPS, the animation is over-budget.
