Critical Findings

Finding 1: Missing HTML Companion File

Medium — File system — Subject Frameworks/History Grade 5/

All other subject frameworks ship with both .md AND .html versions. The History framework is missing its HTML render.

Recommendation: Generate HTML companion file matching the styling of other framework HTMLs (Science, Tarbiya, Tasviriy Sanat, Texnologiya, Geografiya).

Finding 2: PISA Dual-Domain Mapping — Incomplete Implementation

High — §3 (PISA History mapping)

The framework correctly states History uses dual-domain PISA mapping (Reading + Creative Thinking per UNIFIED §7.4). However:

  1. Domain tag inconsistency: §3 says dual-domain (Reading + Social Sciences), but the tagging schema (§15) only shows "domain": "Reading" — no Creative Thinking or Social Sciences sub-domain reference.
  2. No conversion logic: The framework doesn't specify how History performance feeds into the student's Reading PISA profile.
  3. Internal contradiction: §15 shows "domain": "Reading" as a single domain, but §3 declared dual-domain.
Recommendation:
  1. In §15 tagging schema, use "domain": "Reading", "sub_domain": "Social Sciences" OR add a second "domain": "Creative Thinking" entry.
  2. Add explicit student profile tracking structure for History (dual-domain).
  3. Clarify whether "Social Sciences" is a sub-domain or a separate domain.

Finding 3: Custom Track Pattern Not Defined

Medium — §3, §18

Per QWEN.md meta-rule, every non-PISA-pure subject needs its own custom reasoning track. History is missing its Historical Reasoning (HR) track.

Recommendation: Define the Historical Reasoning (HR) track with 4 levels:
  • HR-L1: Recall and identify historical facts, figures, dates
  • HR-L2: Explain cause-and-effect in historical events; classify sources
  • HR-L3: Interpret and compare multiple historical sources/perspectives
  • HR-L4: Synthesize evidence to construct historical arguments

Finding 4: Session Duration — Contradiction with UNIFIED

Medium — §4.1 vs. UNIFIED §4.1

ParameterUNIFIED v2.0History G5
Standard session total30-45 min30-40 min (includes 0-A + 0-B + 7 phases)

Phase calculations sum to 28-44 min. The History 30-40 min cap requires running at the faster end of every phase.

Recommendation: Either align to UNIFIED's 30-45 min range, or add explicit per-phase timing caps that guarantee the 30-40 min total is achievable.

Finding 5: Scholar Pairing Rule — Not Fully Specified

Low — §7.1, §18

§7.1 says "At least 1 Central Asian historical figure per 5 sessions" but the scholar pairing rule from Geografiya G5 (every world figure paired with a Central Asian counterpart) is not fully implemented.

Recommendation: Specify the pairing pattern explicitly:
  • Herodotus ↔ Al-Biruni (historical methodology)
  • Ptolemy ↔ Ulugh Beg (astronomy/cartography)
  • Caesar ↔ Amir Temur (state-building/military leadership)
  • Egyptian calendar-makers ↔ Al-Khwarizmi (mathematical systems)

Finding 6: Missing Framework Shape Declaration

Low — Document structure

Per QWEN.md, every subject framework should declare which of the three shapes it follows. The History framework declares a tiered Boss with HP (§6.6), mapping to PISA-rigorous shape, but this is not explicitly stated.

Recommendation: Explicitly declare the shape at the top of the framework: "PISA-rigorous with Reading-domain anchoring."

Finding 7: Game Break Slot 3 Rule — Timeline Builder Frequency

Low — §6.3

Timeline Builder is a History-specific mechanic (not in either catalog). Its slot assignment needs clarification on which catalog it substitutes from.

Recommendation: Clarify that Timeline Builder counts as a Default 16 substitute for History, or add it to the Default catalog as a domain-specific variant.

Finding 8: Version Number — Should Reflect Iteration Status

Low — Header

Framework declares v1.0 but §18 lists 7 open questions. The status says "Draft specification — first iteration."

Recommendation: Add Status: DRAFT — 7 open items pending to the header.
