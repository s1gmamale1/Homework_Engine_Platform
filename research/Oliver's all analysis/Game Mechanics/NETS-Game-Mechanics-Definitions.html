NETS Game Mechanics — Universal Definitions v1.0

Date: 2026-04-11 Scope: 7 of 28 mechanics Status: Skeleton / v1.0 draft Best-guess fields: PISA, Bloom, cost, time

Purpose. Universal K-11 definitions for 7 game mechanics: Memory Palace, Story Mode, Adaptive Quiz, Mystery Box, Memory Match Blitz, Reaction Chain, Word Ladder Climb. Each mechanic has grade-band design variations and a 15-field spec.

Sources. Primary: NETS-Game-Mechanics-Research-Answers.md (research-backed, cited). Secondary: NETS-Homework-Engine-UNIFIED.md v2.0, NETS-Game-Catalog-Summary.md, G5 psychology findings. Plus 4 design merge decisions from conversation 2026-04-11.

Merge Decisions Applied

  1. Adaptive Quiz round length → pure fixed, no early termination. Predictability > measurement precision for homework.
  2. Memory Match Blitz wrong-confirmation → 3-second correct-answer flash is mandatory. Implementation must not re-hide instantly.
  3. Reaction Chain custom visuals → custom process-diagram visuals only for top-20 chains after 6 months of production data.
  4. Word Ladder Climb rung count → capped at 5-6 rungs max. Each rung is 2 interactions so longer ladders drag.

1. Memory Palace

Place concepts at spatial locations, recall via free recall + temporal order, SM-2 spaced repetition
▼
1.1 Summary

Student places 3-15 concepts (grade-dependent) at distinct locations in a culturally familiar 2.5D scene, then retrieves them via free recall and temporal-order prompts. Palace persists across sessions via SM-2 spaced repetition. One palace per topic to prevent cross-topic interference.

1.2 Core loop
  1. Pick palace location from pool: Registan, Chorsu Bazaar, hovli courtyard, school, own classroom (rural fallback).
  2. System presents N concepts.
  3. For each, tap a hotspot, pick from 2-3 visual images, drag to lock.
  4. Guided walkthrough animates a walk through all hotspots.
  5. Primary recall: free recall (spoken or typed) — "Walk through your palace."
  6. Secondary: temporal-order probes (2-3).
  7. Palace saves. SM-2 schedules revisits.

  • 2-3 authored visual image candidates per concept

  • Hotspot coordinates per background

1.5 Evaluation & retry rules

  • Free recall: Tier 2 semantic match; partial credit per concept. Missing = "Hali emas" (not failed).

2. Story Mode

Human-authored narrative + AI-personalized variants, 30-150s segments, mixed MC + short-answer gates
▼
2.1 Summary

Textbook content delivered as linear narrative with human-authored master stories and AI-personalized variants. Segments 30-150s (grade-dependent) with mixed MC + short-answer comprehension gates. Also the universal Phase 2 driver of every session.

2.2 Core loop
  1. Segment title + illustration.
  2. Text plays (TTS optional for struggling readers).
  3. 2-3 micro-choice points per story — same content, different framing (POV shifts).
  4. Gate between segments: 2 MC (misconception distractors) + 1 short-answer (keyword + semantic).
  5. Gate ≥70% → advance. <70% → remediation branch, retry (max 2).
  6. After 2 failures → "Hali emas", advance anyway, flag teacher.
  7. 4-beat arc: Problem → Struggle → Discovery → Solution.
  8. Stranger Test mandatory: a newcomer must be able to follow.

  • 2-3 micro-choice branch variants
  • Illustrations per segment

  • Remediation branch content

  • MC: immediate feedback, no retry (item pool rotates)

  • Gate: 70% threshold, max 2 retries then advance w/ "Hali emas"

3. Adaptive Quiz

IRT-calibrated item bank + productive-failure scaffolding. Fixed question count, no early termination.
▼
3.1 Summary

IRT-calibrated question bank serves items at student's current theta (ability estimate) with productive-failure scaffolding on wrong answers. Fixed question count — no early termination.

3.2 Core loop
  1. Pick item at current theta.
  2. Student answers. Immediate correct/wrong feedback.
  3. On wrong → Productive Failure scaffold:
    • Step 1: Metacognitive prompt ("What do you know about this?")
    • Step 2: Partial worked solution (first step, not answer)
    • Step 3: Answer + explanation contrasting misconception
  4. Theta updates via Bayesian update.
  5. Next item at theta ± 1 level.
  6. Round ends at fixed question count. Final theta logged.
3.3 Grade-band variations
GradeQuestions/roundFeedbackScaffold depth
K-G23Immediate visual1-step: show correct answer
G3-44Immediate2-step: hint → answer
G5-65Immediate3-step productive failure
G7-97ImmediateFull + misconception contrast
G10-117Immediate + round summaryFull + self-explanation
MERGE APPLIED: Fully fixed question count per grade band. No early termination. The research doc's suggested early-stop on 3 consecutive correct at ceiling would create session-time variance, contradicting its own "predictability > precision for homework" argument.
3.4 Content author input
  • Item bank with IRT parameters (difficulty, discrimination) — pre-calibrated or seeded + live Bayesian update after ~100 responses
  • Metacognitive prompts per item/topic
  • Partial worked solutions per item
  • Misconception-to-correct-reasoning explanations
3.5 Evaluation & retry rules
  • Binary correct/wrong per item
  • No retry on same item in the round — item is "spent"
  • Wrong items re-enter spaced-repetition queue for Memory Sprint
  • Productive failure scaffold triggers automatically on wrong
3.6 PISA coverage

Levels: L1-L6 (full adaptive range). Domains: Math, Science, Reading, languages. Processes: All.

3.7 Bloom's band

Primary: Apply, Analyze. Secondary: Remember, Understand at L1-L2.

3.8 Transition skills

Flexible workhorse — can scaffold any transition depending on item bank. Strongest for L2→L3 ("apply procedure to unfamiliar context") and L3→L4 ("analyze multi-step problem and select approach").

3.9 Grade eligibility
K-G11 universal (scaffold depth scales by grade).
3.10 Subject eligibility & framework_bans

Eligible: Math, Science, Reading, languages, most content subjects.

Banned: framework_bans: ["science_grade5"] — Science G5 bans because it "feels like endless testing" to Uzbek G5 students.

Flagged for review: Tarbiya (no objective right answers), Tasviriy Sanat (creative, not quiz-able).

3.11 Anti-cheat surface

Tier 2 watches: answer time <3s (guessing without reading), pattern answering, theta curve too steep (external help), clipboard paste.

3.12 Telemetry events

aq_item_served, aq_item_answered, aq_scaffold_step, aq_round_complete

3.13 Cost profile

T1: item selection, IRT update, scoring, scaffold selection. T2: SA eval if present, scaffold personalization. T3: none. Per-round: <$0.003

3.14 Device requirements

Low-tier Android. Minimal bandwidth. No special hardware.

4. Mystery Box

Interleaved practice via vague-labeled boxes. NOT the same as Treasure Chest (separate metagame mechanic).
▼
4.1 Summary

Student picks from 2-5 boxes with vague thematic labels ("Something about plants..."). Each box contains a problem from a different prior chapter. Labels sit at the inverted-U curiosity peak — not blind (anxiety), not revealed (boring). Not to be confused with Treasure Chest — see cross-cutting notes.

4.2 Core loop
  1. Present N closed boxes with vague thematic labels.
  2. Student picks any box in any order (autonomy).
  3. Box opens, problem appears.
  4. Identification step (G3+): student identifies problem type (discriminative contrast).
  5. Student solves the problem.
  6. Correct = box claimed; wrong = one retry.
  7. Round ends when all boxes resolved.
  8. 1-in-10-to-15 rounds: bonus "cross-subject connection" box, explicitly framed as bonus.
4.3 Grade-band variations

G3-43Short vague textOptional
G5-63Vague thematicRequired
G7-94Vague thematicRequired + justify
G10-114-5Abstract thematicRequired + justify + connect
4.4 Content author input
  • Problem pool from prior chapters, tagged by category
  • Vague thematic labels per category (noticeable but not revealing)
  • Difficulty levels matched to theta
  • Cross-subject bonus connection problems (flagged)
4.5 Evaluation & retry rules
  • Identification: binary, one retry, no penalty (discriminative contrast is the point)
  • Solve: inherits from item type (MC, SA, etc.)
  • Box order: any — autonomy preserved (SDT)
4.6 PISA coverage

Levels: L2-L5. Domains: All. Processes: Recognize problem type, select strategy, apply prior learning.

4.7 Bloom's band

Primary: Apply, Analyze (strategy selection). Secondary: Remember (prior chapter retrieval).

4.8 Transition skills
  • L2→L3: Recognize when to apply which tool
  • L3→L4: Strategy selection across plausible approaches
4.9 Grade eligibility
K-G11 universal. Identification step scales by grade.
4.10 Subject eligibility & framework_bans

Eligible: all subjects with accumulated chapter history. First-chapter sessions cannot use Mystery Box — no prior chapters to interleave. framework_bans: []

4.11 Anti-cheat surface

Tier 2 watches: always picking same-position box (gaming order), identification answered before box fully opens (automation).

4.12 Telemetry events

5. Memory Match Blitz

Concentration grid + pair-tied confirmation question. Wrong = mandatory 3s correct-answer flash.
▼
5.1 Summary

Concentration-style grid. When a pair is found, a confirmation question tied to that specific pair gates whether it locks. Wrong confirmation → correct answer flashes for exactly 3 seconds (mandatory) → cards re-hide. Leverages hypercorrection effect (Butterfield & Metcalfe 2006).

5.2 Core loop
  1. Grid face-down. Cards are term-definition pairs (mixed).
  2. Student flips two cards.
  3. No match → both re-hide.
  4. Match → confirmation question appears, tied to the matched concept.
  5. Right: pair locks, stays face-up, score increases.
  6. Wrong: correct answer flashes for exactly 3 seconds (mandatory hypercorrection window), then cards re-hide.
  7. Round continues until all pairs locked or soft timer expires.
  8. Final score = pairs locked + speed bonus.
5.3 Grade-band variations
GradeGridPairsTimerConfirmation Q
K-G22×33NoneSkipped
G3-43×46Soft 2 minSimple MC
G5-63×4 or 4×46-8Soft 90sMC + explanation
G7-94×48Soft 90sShort-answer
G10-114×510Soft 60sSA + connection

Max grid: 4×5 (20 cards, 10 pairs). Larger exceeds working memory (~4 chunks per Cowan 2001).

MERGE APPLIED: The 3-second correct-answer flash on wrong confirmation is mandatory, not optional. Implementation must not re-hide cards instantly. The 3-second window preserves the hypercorrection benefit.
5.4 Content author input
  • Term-definition pair pool per chapter
  • Confirmation question per pair — tied to that specific concept, not pool-random
  • Correct-answer flash content (what appears during the 3s window)
5.5 Evaluation & retry rules
  • Match step: no gating
  • Confirmation wrong → mandatory 3s flash → re-hide
  • Pair can be re-attempted in same round (retrieval practice + corrective feedback)
  • Soft timer: points decrease over time, round does NOT end on timeout
  • Speed bonus rewards fluency without punishing slower students
5.6 PISA coverage

Levels: L1-L3. Domains: All with term-definition content. Processes: Retrieve from LTM, map term to meaning.

5.7 Bloom's band

Primary: Remember, Understand.

5.8 Transition skills
  • L1→L2: Map term to definition in context
  • L2→L3: Recognize term in varied form
5.9 Grade eligibility
K-G11 universal (grid size scales by working memory).
5.10 Subject eligibility & framework_bans

Eligible: subjects with term-definition content (Science, Languages, Tarbiya vocab, Geografiya place-names). Less useful for process-heavy subjects but not banned. framework_bans: []

5.11 Anti-cheat surface

Tier 2 watches: confirmation answered faster than human read time (<1s), wrong-then-right patterns (memorizing position, not content).

5.12 Telemetry events

mmb_round_started, mmb_pair_found, mmb_confirmation_result, mmb_hypercorrection_shown, mmb_round_complete

5.13 Cost profile

T1: grid gen, match detection, 3s flash rendering. T2: SA confirmation eval. T3: none. Per-round: <$0.002

Low-tier Android. Minimal bandwidth. No special hardware.

6. Reaction Chain

Conceptually sequential nodes, streak-reset w/ partial credit, top-20 chains get custom visuals
▼
6.1 Summary

4-10 conceptually sequential nodes light up as student answers correctly. 3 wrong in a row resets streak to 0 (lit nodes keep partial credit). Soft 15-30s timer per node. Default visual is linear; top-20 chains get custom process-diagram visuals.

6.2 Core loop
  1. Present N sequential nodes on a topic with natural order (photosynthesis, water cycle, historical chain, algorithm).
  2. Node 1 question appears.
  3. Correct → node lights up, advance. Wrong/timeout → adds to 3-in-row counter.
  4. 3 wrong in a row → chain breaks visually, streak resets to 0, lit nodes keep their points.
  5. Recovery scaffold: "Your streak was [X]. Let's rebuild." Next question is 1 level easier.
  6. Round ends at all-nodes-lit or max question count.
6.3 Grade-band variations

G7-98-1015s soft3 wrong in a row

MERGE APPLIED: Custom process-diagram visuals only for the top 20 chains by usage after 6 months of production data. Everything else uses auto-generated linear-node visual. Stops endless "should we build custom X?" debates.
6.4 Content author input
  • N conceptually sequential concepts in strict order
  • Question per node (calibrated to student level)
  • Recovery question pool (easier than normal)
  • Default linear visual auto-generated; custom diagram authored only for top-20
6.5 Evaluation & retry rules
  • Binary correct/wrong per node
  • Timeout counts as wrong (contributes to 3-in-a-row counter)
  • On break: lit nodes keep partial credit, streak resets, recovery scaffold activates
  • Round completes at all-lit or max question count
6.6 PISA coverage

Levels: L2-L4 (sequential reasoning, process understanding). Domains: Science processes, Math procedures, History timelines, IT algorithms. Processes: Sequence steps, identify causal chain, apply procedural knowledge.

6.7 Bloom's band

6.8 Transition skills

  • L3→L4: Understand causal/procedural flow between steps
6.9 Grade eligibility

7. Word Ladder Climb

Letter-swap word transformation, 2-phase rungs (question + puzzle), LANGUAGE SUBJECTS ONLY
▼
7.1 Summary

Transform start word into target word one letter at a time, each intermediate must be a real dictionary word. Each rung = 2 phases: (1) answer subject question, (2) solve letter-change puzzle as "reward." Language subjects ONLY. Ladder capped at 5-6 rungs max.

7.2 Core loop
  1. Present start word and target word (e.g., COLD → WARM).
  2. Rung 1:
    • Phase A: Subject question (grammar, vocab, definition). Correct → unlocks puzzle. Wrong → hint.
    • Phase B: Perform letter change — 1 letter swap, result must be real dictionary word.
    • Correct change → advance. Wrong → 2 attempts, then hint or auto-skip.
  3. Repeat for 4-5 more rungs until target reached.
  4. Skip budget: 1 skip per ladder total.

MERGE APPLIED: Rung count capped at 5-6 max. Each rung is 2 interactions (question + puzzle) — 6 rungs × 2 = 12 interactions is already at the limit for a Phase 3 game.
7.4 Content author input
  • Pre-validated word ladder database (start/target with known valid paths)
  • Subject question per rung
  • Hint content per rung (first/last letter, definition clue)
  • Dictionary validation source per language
7.5 Evaluation & retry rules
  • Subject Q: correct advances; wrong = hint; 2nd wrong = auto-reveal + advance
  • Letter change: 2 attempts → hint → 1 more attempt → auto-skip
  • Skip budget: 1 per ladder
  • No streak, no hypercorrection — linear progression
7.6 PISA coverage

Levels: L1-L3 (vocabulary recognition, morphological awareness). Domains: Reading primary. Processes: Recognize word form, apply spelling patterns, build vocabulary.

7.7 Bloom's band

Primary: Remember, Apply. Secondary: Understand (word families, morphology).

7.8 Transition skills
  • L1→L2: Recognize word transformations
  • L2→L3: Apply orthographic patterns to new words
7.9 Grade eligibility
G3-G11 only. NOT eligible for K-G2 (pre-reading / early reading stage).
7.10 Subject eligibility & framework_bans

Eligible: Language subjects ONLY — Uzbek Tili, Rus Tili, Ingliz Tili.

Not eligible: Science, Math, Biology, History, Tarbiya, Tasviriy Sanat, Texnologiya, Geografiya, IT, Music — essentially every non-language subject.
framework_bans: ["science_grade5", "tarbiya_grade5", "tasviriy_sanat_grade5", "texnologiya_grade5", "geografiya_grade5"]

03 Cross-Cutting Notes

A. Mystery Box split → Treasure Chest (separate mechanic)

The original catalog conflated two different things under "Mystery Box":

AspectMystery Box (#4 here)Treasure Chest (separate)
PurposePhase 3 interleaved-practice learning mechanicBilim Bazasi metagame reward event
PsychologyRohrer interleaving + Loewenstein curiosityVariable-ratio reward (Skinner)
Research baseInterleaving studiesBehavioral economics
K-11 concernNoneGambling-like mechanic — ethical flag

Recommendation: Define Treasure Chest separately as part of Bilim Bazasi metagame (Layer 4) work. Do not share a name, UI, or codebase.

B. Word Ladder → Suffix Ladder variant for Uzbek

Classic letter-substitution word ladders are an English-specific mechanic. They don't translate to agglutinative Turkic languages. A sibling mechanic, Suffix Ladder, should be designed for Uzbek Tili and Rus Tili:

  • Add or remove one suffix per step
  • Each step must be a real inflected/derived word
  • Validated against a morphological analyzer (MorphUz or equivalent)
  • Same pedagogy (morphological awareness), appropriate implementation

Action: Flagged for linguistics review and separate mechanic definition when language subject frameworks are written.

C. The framework_bans pattern

Every mechanic definition includes a framework_bans field — explicit list of subject-grade frameworks that ban this mechanic, with rationale. Makes ban decisions visible to designers instead of hidden in override docs. Grows into a cross-subject compatibility matrix over time.
