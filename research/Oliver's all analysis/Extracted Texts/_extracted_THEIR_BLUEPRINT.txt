## The 3 Sacred Criteria (Every Decision Must Pass All 3)

## The Engagement Philosophy
We use White Hat gamification (epic meaning, accomplishment, empowerment) as primary drivers and Black Hat gamification (scarcity, loss avoidance, unpredictability) sparingly and transparently. Per NETS Principle 3: we celebrate LEARNING progress, not compliance. Zero pay-to-win. Zero real-money purchases.
Frameworks integrated:
Hook Model (Nir Eyal): Trigger → Action → Variable Reward → Investment — one full cycle per session
Octalysis (Yu-kai Chou): 8 Core Drives mapped to specific phases
Compulsion Loop: Anticipation → Action → Reward — dopamine hits at variable intervals
Duolingo Mechanics: Streaks, XP leaderboards, hearts system (as hint tax), streak freeze, leagues, reward chests, legendary challenges — adapted for education

What AI loads:

pages: "6-8",

Verification gate: Content pool for this topic must have ≥85 tagged items (SPEC §10.2). If insufficient → flag to teacher, session CANNOT proceed. This is a hard block, not a soft warning.
Failure mode: If assignment data is missing → show student "Homework not yet available. Contact your teacher." No partial sessions.
## 0.3 Difficulty Band Calculation
Inputs:
Student's current PISA level in this subject
Grade's target PISA level
Teacher overrides (if any — SPEC §14)
Student's weakest domain areas
Algorithm:
difficulty_band = {
floor: max(student_pisa - 1, 0),              // Never below PISA 0
target: student_pisa,                          // Most items at current level
ceiling: min(student_pisa + 1, 6),             // Stretch items one level above
teacher_override: teacher_floor || teacher_ceiling || null
}
Output determines: Which items from the pool get selected, how hard the boss questions will be, which game mechanics are appropriate, which transition skills to target.
## 0.4 Session Assembly
[AI: Tier 1] selects specific content items from the pre-generated pool for each phase:
session_plan = {
phase_1_memory_sprint: SELECT 5-8 recall_items
FROM previous_chapters
WHERE spaced_repetition_due = true
ORDER BY priority DESC                       // Wrong > approaching_threshold > stale

phase_2_story_mode: SELECT 3 narrative_segments
FROM this_chapter
WHERE standard_ref IN assignment.learning_objectives

phase_3_game_breaks: SELECT 3 game_items
WHERE at_least_1_reinforcement                // At student's current level
AND at_least_1_stretch                        // One level above
AND at_least_1_transition_skill_match          // Targets in-progress skill
AND no_mechanic_repeats_more_than_2x

phase_4_real_life: SELECT 1 transfer_scenario
WHERE pisa_level >= 2                         // Below L2 → extra adaptive quiz instead
AND cultural_context = "uzbek"

phase_5_consolidation: SELECT 1 mnemonic_exercise
WHERE technique matches grade_band

phase_6_boss: SELECT 5-7 boss_questions
WHERE 40% easy + 40% medium + 20% hard        // SPEC §5.6 line 691
AND covers_at_least_2_transition_skills

phase_7_reflection: SELECT 1 reflection_prompt
// Will be dynamically adjusted based on performance
}
Transition skill coverage verification: At least 2 different transition skills must be targeted across the session. At least 1 must match the student's in_progress skills. If this fails → re-select items until constraint is met.
## 0.5 Lesson Brief Generation
[AI: Tier 1] selects (or Tier 2 generates) a 4–5 bullet point summary of the lesson.
Content derived directly from textbook key definitions ("Yodda tuting!" boxes, chapter opening paragraphs)
This summary will flash ONCE at session start and never appear again — it's a reference anchor, not a crutch
Creates urgency to actually read the textbook (if student didn't read it, this is their only preview)

# SECTION 1: PHASE 1 — ENGAGE + REMEMBER (Memory Sprint)
Bloom's Level: Foundation → Remember
PISA Level: Below 1 → Level 1
Duration: 2–3 minutes
AI Tier: [AI: Tier 1] — Pre-generated items, algorithmic selection
SPEC Reference: UNIFIED §5.1
Octalysis Drives: Accomplishment (XP), Empowerment (speed control), Unpredictability (variable XP)
This phase has one job: pull the student IN. The first 30 seconds determine whether they engage or quit.
## 1.1 The Hook Game (30 seconds)
[HOOK: Action — Low Friction] A fast-tap game that is intentionally EASY and ADDICTIVE. XP and streaks start accumulating immediately.
What the student sees: A rapid-fire classification game. Items appear on screen, student taps the correct category. No typing. No thinking. Pure reflex + recall.
Subject-specific hooks:

Psychological principle: Csikszentmihalyi's flow state — start with LOW challenge / HIGH reward to pull the student in, then gradually increase challenge. The hook game is NOT assessment. It's momentum.
AI selects game type based on subject + student's weakest domain. Sets initial difficulty LOW — 90%+ success rate expected.
What AI tracks: Response speed, accuracy, which categories cause hesitation — this data feeds ALL later phases.
Scoring:
50 XP per correct tap
Speed bonus: +10 XP if answered in <2 seconds
[HOOK: Variable Reward] Random "SUPER!" animation on ~20% of correct answers → +25 bonus XP (variable ratio reinforcement — Skinner)
10-second timer, 8–12 items

## 1.3 Flashcard Sprint (Remember Gate)
Purpose: Test recall of PREVIOUS chapters. Not current topic. This is spaced repetition in action.
Parameters:

Item Selection Algorithm (SM-2 Modified):
1. GET all recall_items from chapters BEFORE current chapter
2. PRIORITIZE by forgetting curve:
Priority 1: Items student got WRONG in previous sessions
Priority 2: Items approaching forgetting threshold (Ebbinghaus: retention < 60%)
Priority 3: Items answered correctly but not seen in 7+ days
Priority 4: Items answered correctly, seen recently (filler if needed)
3. FILTER to target count, balanced across prior chapters
4. RANDOMIZE order within priority bands
Scoring:
Correct: +100 XP
Streak bonus: +10 XP × current_streak_within_sprint (resets per sprint)
Speed bonus: +10 XP if answered in <5 seconds
[HOOK: Variable Reward] "MUKAMMAL!" animation on 3+ streak → bonus XP varies (50–150, randomly)
No penalty for wrong answers — this is warm-up, not assessment
Adaptation Gate:
IF accuracy >= 80%  → Student ready for new content. Proceed to Phase 2.
IF accuracy 60–79%  → Proceed but FLAG for teacher review. AI notes which standards decayed.
IF accuracy < 60%   → ROUTE TO REMEDIATION before new content begins.
System inserts a 2-minute micro-review targeting the failed standards.
Then proceeds to Phase 2 with adjusted difficulty band (floor lowered by 0.5).
Standards reference: Every recall item displays its standard code in the post-session review: UZ-MATH-5-NATNUM-03 — Natural sonlarni yaxlitlash
Research basis: Retrieval Practice (Roediger & Karpicke, 2006) — active recall is 3x more effective than re-reading. Forgetting Curve (Ebbinghaus, 1885) — without spaced review, 70% of learning is lost within 24 hours.

## 2.3 Narrative Cultural Framework
[HOOK: Epic Meaning — Octalysis Core Drive 1] The student is the HERO of the story. They're not passively reading — they're discovering, exploring, solving.

Character names: Always Uzbek — Sardor, Malika, Alisher, Nargiza, Jasur, Dilnoza
Settings: Always real Uzbek locations — Samarkand, Bukhara, Tashkent, Khiva, Fergana Valley, student's own mahalla
Currency: Always so'm (never USD or abstract units)
## 2.4 Checkpoint Questions
Purpose: Comprehension gates, NOT assessment. They check "did you understand what you just read?"
Rules:
1–2 questions between each segment
Student MUST answer correctly to proceed to the next segment
On wrong answer: show explanation referencing the EXACT textbook page, then re-ask
Maximum 2 retries per checkpoint. After 2 failures: [AI: Tier 2] provides a simplified version of the question with scaffolding
Format: varies by grade (MC for Grades 1–2, type answer for 3+)
Skimming detection: AI tracks time per segment. If student spends <30 seconds on a 90-second segment → flagged as "likely skimmed." This flag increases the checkpoint difficulty slightly and adds an extra Story Mode micro-segment at end if pattern persists.
## 2.5 Book Reference Integration
Every segment shows a persistent 📖 Darslik: sahifa X (Textbook: page X) reference in the corner.
On checkpoint failure: The textbook page reference PULSES and enlarges — a visual nudge to actually read the book. This reinforces Criterion 1: the book is the lesson, NETS is the transformer.
Research basis: Narrative Retention (Willingham, 2004) — stories are retained 22x better than raw facts. CPA approach (Leong et al., 2015) — structured concrete-pictorial-abstract progression for mathematical understanding.

## 3.1 Game Selection Algorithm
[AI: Tier 1] selects 3 games per session:
INPUTS:
- topic.learning_objectives          // What concepts to practice
- student.pisa_level                 // Current ability
- student.weakest_domain             // Where they struggle
- student.in_progress_skills         // Transition skills being developed

SELECTION RULES:
Rule 1: At least 1 REINFORCEMENT game (at student's current Bloom's/PISA level)

### Game: Tile Match
What student sees: Grid of 8–12 face-down tiles (4–6 pairs). Tap to reveal, find matches.
Gameplay:
Tap first tile → reveals content (term, image, formula, or concept)
Tap second tile → reveals content
Match found → tiles fade out with celebration animation, +100 XP, timer +3 seconds bonus
No match → tiles shake (3×, 100ms), return face-down, timer −5 seconds
WIN: All pairs matched before timer = 0
LOSS: Timer reaches 0 → partial XP (50 per completed pair)
[HOOK: Variable Reward] On perfect match (first attempt, no wrong taps): random chance (30%) of "AJOYIB!" super-animation + bonus 50 XP. Variable ratio — not every perfect match triggers it.
Difficulty by grade:

Subject examples:
Math: "1/2" ↔ [visual: 1 of 2 parts shaded]
Science: "Chloroplast" ↔ [diagram with chloroplast highlighted]
History: "Moddiy manba" ↔ [image of stone tool]
Reading: "Metafora" ↔ "Sinf xona — hayvonot bog'i edi"
XP: Perfect = 300 | Partial = 100–250 | Streak bonus applies
────────────────────────────────────────────────────────────────────────────────
### Game: Sentence Fill (Cloze Test)
What student sees: Text passage with blanks. Student fills each blank by selecting from word bank (Grades 2–7) or typing (Grades 8–11).
Blank selection rules:
Remove every 3rd content word (Grades 2–4)
Remove every 4th content word (Grades 5–7)
Remove every 5th content word (Grades 8–11)
NEVER remove: proper nouns, numbers, key formulas
ALWAYS keep first and last sentence intact
Word bank:
Grades 2–4: Correct answers + 3 distractors per blank
Grades 5–7: Correct answers + 2 distractors per blank
Grades 8–11: No word bank (free recall)
XP: 100 per correct fill | First-attempt bonus: +25 | All correct: +100 bonus
Timer: 2–3 minutes
────────────────────────────────────────────────────────────────────────────────
### Game: Mystery Box (Interleaving)
What student sees: 3–5 closed boxes with "?" on them. Each contains a problem from a DIFFERENT category (current + previous chapters).
Gameplay:
Tap box → opening animation (anticipation!) → problem revealed
FIRST: Student identifies problem TYPE from dropdown (e.g., "fraction addition" vs "fraction comparison")
THEN: Student solves the problem
Feedback on BOTH type identification AND solution
[HOOK: Unpredictability — Octalysis Core Drive 7] Student doesn't know what's inside each box. The mystery IS the engagement. Curiosity drives completion. Same psychology as loot boxes, but educational.
Scoring:
Correct type ID: +50 XP
Correct solution: +150 XP
Both correct: +50 bonus
Total per box: 250 XP maximum
[HOOK: Near-Miss] If student gets type right but solution wrong: "Juda yaqin! Turi to'g'ri — javobni tekshiring" (So close! Type is correct — check the answer)
Availability: Grades 3+ (not available Grades 1–2)
Research basis: Interleaving (Rohrer & Taylor, 2007) — mixing problem types improves transfer by 43% vs. blocked practice.

### Game: Why Chain (Socratic Dialogue)
What student sees: A "Why?" question. Student answers. AI asks "But WHY?" again. Chain deepens.
AI Dialogue Pattern:
Level 1 (Surface):
AI: "Nima uchun o'simliklar yorug'likka muhtoj?"
Student: "O'sish uchun."
AI: "Yaxshi boshlanish! Lekin NIMA UCHUN yorug'lik ularga o'sishga yordam beradi?"

Level 2 (Mechanism):
Student: "Yorug'lik ularga energiya beradi."
AI: "Ajoyib! O'simliklar bu energiya bilan NIMA qiladi?"

Level 3 (Process):
Student: "Ular CO2 va suvdan oziq moddalar yaratadi."
AI: "Zo'r! Siz fotosintezni tasvirladingiz!"
Chain length: 3–5 levels minimum
Timer: 3 minutes
AI Rules: NEVER give direct answer | ALWAYS acknowledge student's response | ALWAYS probe deeper | Reference textbook pages when student is stuck
PISA level scaling:
Level 2–3: 2–3 chain levels, guided prompts
Level 3–4: 4 levels, less scaffolding
Level 5–6: 5+ levels, open-ended, competing explanations accepted
XP: 150 per chain level completed | All levels: +250 bonus | Hint: −10 XP per hint
AI Tier: [AI: Tier 2] — Requires real-time dialogue management
Note for History: Why Chain requires deeper prior knowledge than introductory lessons provide. Reserve for chapters where students have sufficient context for causal reasoning. (Lesson learned from demo build — see SPEC §7.4)
Research basis: Elaborative Interrogation (Pressley et al., 1992) — "Why?" questions improve reasoning by +2 PISA sub-levels.

### Game: Adaptive Quiz (IRT-Calibrated)
What student sees: Standard quiz format — question, answer input, feedback. But difficulty adapts after each answer.
IRT Implementation:
Each question has pre-calibrated parameters:
- difficulty (theta): -3.0 to +3.0
- discrimination (alpha): 0.5 to 2.0
- guessing (c): 0.1 to 0.25

Algorithm:
1. Start at theta = 0 (grade-level expectation)
2. Present question with difficulty ≈ current theta
3. IF correct: theta += 0.3
IF incorrect: theta -= 0.3
4. Select next question with difficulty ≈ new theta
5. Repeat for 5–7 questions
6. Final theta = student's ability estimate for this standard

PISA mapping:
theta -3 to -1  → PISA Level 1
theta -1 to 0   → PISA Level 2
theta 0 to 1    → PISA Level 3
theta 1 to 2    → PISA Level 4
theta 2 to 3    → PISA Level 5+
XP: Based on final theta achieved (higher theta = more XP)
Timer: 2–3 minutes total

## 3.4 In-Game Adaptation
[AI: Tier 1] monitors performance WITHIN each game and adjusts in real-time:
WITHIN EACH GAME (checked every 3–5 items):

IF success_rate < 60%:
→ difficulty_tier -= 1
→ Enable hints (if disabled)
→ Provide simpler representations
→ DO NOT drop below student's established floor

IF success_rate > 90%:
→ difficulty_tier += 1
→ Reduce hint availability
→ Introduce next Bloom's level items

IF success_rate 70–80%:
→ FLOW ZONE. Maintain current tier.

EMERGENCY (within any single game):
IF accuracy < 40%:
→ Flag the specific standard
→ After this game, insert a Story Mode micro-segment for that standard
→ The micro-segment references the exact textbook page
Research basis: Flow Theory (Csikszentmihalyi, 1990) — maintaining 60–80% success rate = optimal learning zone.

This phase exists because Phase 6 (Boss) is HIGH-STAKES with LIMITED hints. Students need their knowledge consolidated and readily retrievable.
The psychological setup: Student just placed 5 concepts in a mental palace. They feel PREPARED. This confidence reduces test anxiety and improves boss performance. The consolidation phase is the "I'm ready" moment before the battle.
Research basis: Method of Loci (Maguire et al., 2003) — 300% retention improvement when concepts are spatially anchored.

# SECTION 6: PHASE 6 — EVALUATE + CREATE (AI Boss Fight)
Bloom's Level: Apply → Analyze → Evaluate → Create (tiered by question difficulty)
PISA Level: Level 3–6 (tiered)
Duration: 5–10 minutes (standard) / 10–15 minutes (extended)
AI Tier: [AI: Tier 1] for question selection, HP math, combos; [AI: Tier 3] for real-time answer evaluation, Socratic tutoring, question regeneration
SPEC Reference: UNIFIED §5.6
Octalysis Drives: ALL 8 DRIVES ACTIVE — This is the engagement centerpiece
THIS IS THE MANDATORY MASTERY GATE. ZERO EXCEPTIONS. NO SKIPPING.
The student fights an AI Boss that tests deep understanding. Every session ends with a boss. If the boss isn't defeated, the homework isn't done.
## 6.1 Boss Mechanics
HP System:

Damage per correct answer:

Combo system: 3 correct in a row → 2× damage on next answer. This means a student on a hot streak can deal massive damage. [HOOK: Near-Miss] If combo breaks at 2: "Bitta qoldi edi! Keyingi safar!" (One more was needed! Next time!)
Hint tax: Requesting a hint costs HP — the BOSS heals:

This is the Hearts system adapted from Duolingo. Hints are scarce resources. Each hint is a strategic cost — creates tension between "I need help" and "I can figure this out." (Octalysis Core Drive 6: Scarcity)
## 6.2 Question Format by Grade

[AI: Tier 3] evaluates open-ended answers for reasoning quality, not just keyword matching. AI checks against textbook-stated facts — cannot accept factually wrong reasoning even if logically structured.
For non-exact subjects (History, Reading): AI accepts multiple valid interpretations but REQUIRES evidence/reasoning. "I think so" is not accepted; "I think so because sahifa 7da aytilganki..." is accepted.
## 6.3 What Happens: VICTORY
Boss HP reaches 0 → VICTORY SEQUENCE:

1. Boss defeat animation (dramatic, satisfying)
2. Star calculation:
┌─────────────────────────────────────────────────┐
│ ⭐        Boss defeated (any # of attempts)      │
│ ⭐⭐      ≤2 attempts, >50% HP remaining         │
│ ⭐⭐⭐    1st attempt, no hints, >80% HP         │
└─────────────────────────────────────────────────┘

3. [HOOK: Reward Chest — Variable Reward]
Animated treasure chest opens with:
- XP: 500 (1 star) / 700 (2 stars) / 1000 (3 stars)
- Random avatar item: 10% chance (1 star), 25% (2 stars), 50% (3 stars)
- Rare badge progress: only on 3-star victories
The anticipation during the chest-opening animation IS the dopamine hit.

4. [HOOK: Social Influence]
"Sardor bugun 3 yulduz oldi! Siz ham 3 yulduz ola olasizmi?" (class opt-in only)

5. All performance data → Phase 7 (AI Analysis)
## 6.4 What Happens: DEFEAT
Boss HP > 0 after all questions → BOSS WINS:

1. [AI: Tier 3] activates Socratic Tutoring:
- AI identifies which learning objectives the student failed on
- AI maps failures to specific textbook sections and standards
- AI asks GUIDING QUESTIONS — NEVER gives answers directly
- AI references specific textbook pages: "Sahifa 7ga qarang. Moddiy manba nima?"
- AI probes deeper: "Siz moddiy manba haqida bilasiz. Yozma manba nimasi bilan farq qiladi?"

2. AI REGENERATES boss questions on retry:
- Same standards, same PISA levels, same Bloom's levels
- DIFFERENT numbers, contexts, scenarios
- Prevents answer-sharing between students or across attempts

3. Retry limit: Maximum 3 attempts
- Attempt 1 failed → Socratic tutoring → regenerated questions
- Attempt 2 failed → Socratic tutoring (deeper) → regenerated questions
- Attempt 3 failed → Teacher notified with specific weak standards + all response data
Student marked as "needs intervention" for this topic
Session completes with 0 stars (homework technically incomplete)

4. [HOOK: Near-Miss Psychology]
After defeat: "Boss HP: 15 qoldi! Ozgina qoldi edi!" (Boss had 15 HP left! So close!)
This creates the "almost won" tension that drives retry motivation.
(Same psychology as slot machines showing 2 of 3 matching symbols)

## 8.2 Targeted Micro-Exercises
For each identified weak area: 1–2 quick exercises specifically targeting that concept.

Designed to rebuild from the specific failure point
Outcomes:

→ Concept marked as "stabilizing"
→ Added to spaced repetition queue: review in 3 days
→ +100 XP per correct micro-exercise (bonus XP framing)

IF student STILL fails micro-exercise:
→ Concept marked as "persistent weakness"
→ Teacher notified with specific concept + all response data
→ Added to spaced repetition queue: review TOMORROW
→ Will appear prominently in next session's Memory Sprint
