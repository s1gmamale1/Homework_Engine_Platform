## 1. FAMILY DEFINITION

**Aniq Fanlar** (Uzbek: "Exact Sciences") — the family of subjects governed by derivable, verifiable rules. Every concept in this family has a definitive correct answer that can be reached through logical reasoning, formal procedure, or mathematical derivation. Knowledge in this family is cumulative and hierarchical: each new topic builds directly on prior topics, and gaps in foundational knowledge cascade into failure at higher levels.

**Uzbek gloss:** "Aniq" = exact, precise, definite. These are the sciences where there is a right answer and a provable path to it.

**Epistemological commitment:** The textbook is the source of truth. NETS transforms delivery but never alters core concepts, formulas, or derivations. If AI-generated content contradicts the textbook, the textbook wins.

## 2. SUBJECTS IN THIS FAMILY

| Subject | Grades | Uzbek Name | Notes |
|---|---|---|---|
| **Mathematics** | K–11 | Matematika | Core subject. CPA progression mandatory. Foundation for all other exact sciences. |
| **Physics** | G7–11 | Fizika | Applied mathematics + empirical reasoning. Formal notation + experimental design. |
| **Chemistry** | G7–11 | Kimyo | Symbolic notation (periodic table, equations) + procedural reasoning (reactions, stoichiometry). |
| **Informatics** | G5–11 | Informatika | Algorithmic thinking, formal notation (pseudocode, syntax), logic. *Default placement in Aniq Fanlar due to algorithmic demands. ADR required if standalone framework warrants different treatment.* |

**Why these four cluster together:** All four share the same cognitive profile — multi-step calculation, formal notation, derivation from first principles, and error detection. A student strong in Mathematics will transfer skills to Physics and Chemistry more easily than a student strong in Literature. The mechanic emphasis is therefore shared, not reinvented per subject.

## 3. COGNITIVE DEMAND PROFILE

The Aniq Fanlar family targets these cognitive skills, ranked by importance:

| Rank | Cognitive Demand | Description | PISA Mapping |
|---|---|---|---|
| 1 | **Multi-step calculation** | Execute procedures in correct order, track intermediate results | Math: Employ, Interpret |
| 2 | **Formal notation fluency** | Read and write equations, formulas, symbols, algorithmic syntax | Math: Quantity; Science: Knowledge |
| 3 | **Derivation from first principles** | Start from known rules and derive new results (not just memorize) | Math: Apply; Science: Explain |
| 4 | **Error detection** | Identify where in a chain of reasoning or calculation a mistake occurred | All domains: Evaluate |
| 5 | **Concept understanding** | Explain WHY a formula works, not just execute it | Math: Reflect; Science: Use Evidence |
| 6 | **Rule memorization for formula recall** | Rapid retrieval of formulas, constants, identities under time pressure | Math: Remember |
| 7 | **Spatial reasoning** | Visualize geometric transformations, force diagrams, molecular structures | Math: Space & Shape; Science: Interpret |

**Developmental considerations (Piaget):**
- **Grades 3–4 (Concrete Operational):** CPA staging mandatory. Every formula begins with physical objects or visual models before symbolic notation.
- **Grades 5–7 (Transitional):** Concrete scaffolding gradually withdrawn. Students begin abstract reasoning but still need pictorial anchors.
- **Grades 8–11 (Formal Operational):** Full abstract reasoning. Hypothesis testing, proof construction, multi-variable analysis.

## 4. GAME MECHANICS — SUITABILITY MATRIX

### 4.1 PRIMARY MECHANICS (Always Available — Core to This Family)

| Mechanic | Pool | Why It Fits Aniq Fanlar | Primary Phases | PISA/Bloom's Target |
|---|---|---|---|---|
| **Story Mode** | Default | **Core concept delivery mechanism.** Textbook content as narrative with 4-beat arc: Problem → Struggle → Discovery → Solution. CPA staging embedded — every new formula introduced through story context before symbolic abstraction. Stranger Test quality gate. | Phase 2 (Concept Delivery) | L1→L2: Remember→Understand |
| **Tile Match** | Default | Formula assembly, term↔definition pairing, equation completion. Drag-and-drop makes abstract symbols tactile. | Phase 3 (Game Break) | L1→L2: Remember/Understand |
| **Notebook Capture** | Default | **CRITICAL for this family.** Student does derivations, proofs, and diagrams by hand, photographs them. Generation Effect = 25–40% retention gain over typing. | Phase 3 or 5 | L2→L3: Apply/Analyze |
| **Worked Examples** | Default (implicit) | AI presents step-by-step solved example, student fills in missing steps. Sweller's cognitive load reduction. | Phase 2–3 | L2→L3: Understand/Apply |
| **CPA Progression** | Default (structural) | Concrete → Pictorial → Abstract. Every new formula follows this sequence. Non-negotiable for Mathematics. | Phase 2 (Story Mode) | L1→L2: Remember→Understand |
| **Error Analysis Tasks** | Default (implicit) | Deliberately-wrong worked examples presented for student to diagnose. "What mistake did this student make?" | Phase 3 (Game Break) | L3→L4: Analyze/Evaluate |
| **Adaptive Quiz** | Default | IRT-calibrated practice at student's exact proficiency level. Essential for multi-step calculation fluency. | Phase 3 (Game Break) | All levels |
| **Final Boss** | Default | Open numerical/derivation problem. **No multiple-choice for Grade 6+.** Rubric-based partial credit. | Phase 6 | L2–L5: Apply→Create |
| **Speed Match** | Default | Arithmetic fluency drills, formula↔result pairing. Builds automaticity so working memory is freed for reasoning. | Phase 1, Phase 3 | L1: Remember |

### 4.2 SECONDARY MECHANICS (Context-Dependent — Use When Appropriate)

| Mechanic | When to Use | Caveats |
|---|---|---|
| **Why Chain** | Causal reasoning: "Why does this formula work?" **Max 2 levels for G5, 3 levels for G6+.** | Not for pure calculation topics. Best for concept-understanding topics (e.g., "Why does the quadratic formula give two answers?") |
| **Spaced Flashcards** | Formula memorization, constant recall (π, g, periodic table elements). | Use sparingly — flashcards alone produce shallow recall without understanding. Always pair with Tile Match or Notebook Capture. |
| **Memory Sprint** | Phase 1 warm-up from prior chapters. Format: Quick MC, Speed Match, Fill-in-Blanks. | Content must be from PREVIOUS chapters only. Never test current-topic content in Phase 1. |
| **Tic Tac Toe vs AI** | Recall-heavy practice where strategy makes drill feel like a duel. | Best for arithmetic facts, formula identification, quick mental math. |
| **Connect Four vs AI** | Multi-step procedures where planning ahead matters (e.g., solving equations, balancing chemical reactions). | Good for G7+ where students can think 2–3 steps ahead. |
| **Codebreaker** | Deductive reasoning + content recall. Periodic table, math formulas, algorithm syntax. | Works best when the "code" metaphor maps naturally (e.g., decode a chemical formula). |
| **Reaction Chain** | Process sequences: order of operations, algorithm flow, lab safety procedures. | Builds fluency under light pressure. Good for G5–G7. |
| **Mystery Box** | Cross-chapter retention. Student must recognize WHICH tool to use, not just how. | Excellent for topics where students confuse methods (e.g., when to use LCM vs. GCD). |
| **Puzzle Lock (Sliding Tile)** | Spatial reasoning: geometry transformations, map assembly, diagram reconstruction. | Higher render cost — skip on low-end devices. |
| **Real-Life Challenge** | Transfer: apply formulas to real-world scenarios (bazaar pricing, construction, travel). | **Must use first-person expert POV.** Student IS the engineer, accountant, or analyst. |
| **Peer Teaching** | G5–11. Student explains a math concept to a fictional classmate. Forces deep processing. | Tier 2 AI evaluation required. Best for G8+ where explanations are sophisticated. |
| **Memory Palace** | Locking formulas, constants, periodic table groups before Boss fights. | Registan Square default. Best for G5+. Use for lists and classifications, not for derivations. |

### 4.3 BANNED or NOT RECOMMENDED MECHANICS

| Mechanic | Status | Reason |
|---|---|---|
| **Movement Breaks** | Not Recommended for G5+ | Developmentally appropriate for G1–4 only. Aniq Fanlar content for older students requires sustained cognitive effort that physical breaks disrupt. |
| **Sentence Fill (Cloze)** | Rarely Used | Designed for language/grammar. Only appropriate for Aniq Fanlar when filling in terminology in a science definition or algorithm comment. |
| **Word Ladder Climb** | Banned | Purely linguistic mechanic. No natural fit for mathematics, physics, chemistry, or informatics. |
| **Reflection Journal** | Phase 7 Only | Metacognitive value but low practice density. Use only as the session-closer (Phase 7), not as a practice mechanic. |
| **Bridge Builder** | Conditional | Physics and math applications only. Higher render cost — must be skipped on low-end devices. |
| **Minefield Navigator** | Conditional | Spatial reasoning value but higher render cost. Use selectively for geometry, periodic table navigation. |
| **Escape Room: Subject Lock** | Big Boss / Mythical Boss Only | Multi-concept synthesis. Too complex for standard sessions. Reserve for weekly Big Boss or random Mythical Boss events. |
| **Territory Conquest** | Big Boss / Mythical Boss Only | Cross-subject review game. High stakes, strategic depth. Not for daily homework. |

## 5. BUZAN TECHNIQUES USED IN ANIQ FANLAR

Tony Buzan's cognitive science research underpins the NETS session design. The following Buzan techniques are specifically applied to the Aniq Fanlar family:

### 5.1 Core Buzan Techniques (Applied Across All Phases)

| Buzan Technique | NETS Implementation | Where in Session | Aniq Fanlar-Specific Application |
|---|---|---|---|
| **BOST (Buzan Organic Study Technique)** | Browse → Prime → Overview → Inview → Review | Phases 0-A (Theme Preview) → 0-B (Flash Cards) → 2–3 (Story + Games) → 5+7 (Consolidation + Reflection) | Student scans the chapter structure (0-A), primes existing knowledge ("What formulas do you already know?"), overviews key cards (0-B), engages with content (2–3), reviews (5+7). This is the full study cycle modeled for every homework. |
| **MIG (Most Important Graph — Recall Curve)** | Recall peaks at session start (Primacy) and end (Recency). Dips mid-session. | Phase 1 (Memory Sprint) = Primacy peak. Phase 5 (Consolidation) = Recency peak. | In Aniq Fanlar, the most important prior-knowledge formulas are tested in Phase 1 (Primacy). New formulas are introduced in Phase 2–3 (the dip, where engagement is lowest), then re-locked in Phase 5 (Recency). This matches the brain's natural attention pattern. |
| **Dual Coding** | Every concept presented with visual + verbal representation simultaneously. | Phase 2 (Story Mode), Phase 3 (Tile Match), Phase 5 (Memory Palace) | Formulas are ALWAYS paired with a visual model (bar model for fractions, force diagram for physics, molecular model for chemistry). Text-only formula presentation is banned. |
| **Radiant Thinking** | Concepts branch from a central idea, not linear lists. | Phase 2 (Story Mode structure), Phase 5 (Mind maps) | When introducing a new formula (e.g., quadratic formula), the story radiates: historical context → visual model → symbolic form → real-world application. Not: "Here is the formula. Memorize it." |
| **80/20 Keyword Tagging** | Identify the 20% of terms that carry 80% of meaning. | Phase 3 (Sentence Fill — when used), Phase 5 (Flash Cards) | In Aniq Fanlar: the critical keywords are the formal terms (e.g., "denominator," "coefficient," "velocity"). These are flagged in Flash Cards and tested in Sentence Fill when appropriate. |
| **Link System (Mnemonic Chaining)** | Concepts linked into a memorable chain through association. | Phase 1 (Link Chain format for Memory Sprint), Phase 5 (Memory Palace) | Example: To remember the order of operations (PEMDAS), the student creates a story: "Please (Parentheses) Excuse (Exponents) My (Multiplication) Dear (Division) Aunt (Addition) Sally (Subtraction)." The chain is tested in Memory Sprint Link Chain format. |
| **Primacy–Recency Effect** | Memory is strongest at the start and end of a session. | Phase 1 (Primacy) → Phase 5+7 (Recency) | The most important prior-knowledge items (formulas the student previously struggled with) are placed in Phase 1. The new topic's core formula is re-tested in Phase 5 and Phase 7. The mid-session dip (Phase 3) is filled with game mechanics that require less working memory. |
| **Von Restorff Effect** | Distinctive items are remembered better. | Phase 3 (Game Breaks — novelty), Phase 6 (Boss — visual distinctiveness) | When engagement dips, a visually distinctive game mechanic (e.g., Puzzle Lock instead of standard Tile Match) creates a Von Restorff anchor. The Boss fight has a distinct visual theme (dark background, dramatic music) that makes it memorable. |
| **Cortical Diversity** | Sessions must engage both left-brain (logical, sequential) and right-brain (visual, creative) skills. | Every session must include both types | **Left-brain mechanics:** Adaptive Quiz, Why Chain, Speed Match, Codebreaker. **Right-brain mechanics:** Tile Match (visual), Memory Palace (spatial), Notebook Capture (drawing), Puzzle Lock (spatial). A session with ONLY Quiz + Why Chain fails the Cortical Diversity constraint. |
| **Peg System** | Numbers/concepts pegged to pre-memorized image anchors. | Phase 5 (Memory Palace — optional extension) | For G8–11: students peg formulas to familiar locations. Example: the quadratic formula is "pegged" to the Registan archway — each term of the formula corresponds to an architectural feature. |
| **Buzan Review Intervals** | 10 min → 24 h → 1 week → 1 month. Spaced retrieval prevents forgetting. | Phase 1 (Memory Sprint item selection), Phase 5 (Consolidation) | The spaced repetition algorithm (SM-2) is constrained by Buzan Review Intervals as a floor: no concept can skip review if it hasn't been seen within the Buzan interval, even if SM-2 says it's "mastered." This is critical for Aniq Fanlar because formula forgetting cascades into topic failure. |

### 5.2 Buzan Techniques NOT Applied to Aniq Fanlar

| Technique | Why Not Applied |
|---|---|
| **Mind Mapping (as primary practice)** | Mind maps are excellent for brainstorming and overview but produce low practice density for exact sciences. Used only optionally in Phase 5 for G5–G7. Replaced by Notebook Capture for older students. |
| **Speed Reading** | Not applicable. Aniq Fanlar requires careful, deliberate reading of problems — speed reading produces careless errors. |
| **Creative Story Linking (beyond Link System)** | Extended creative storytelling is a time sink for exact sciences. The Link System (mnemonic chaining) is sufficient. Full Story Mode is reserved for Phase 2 concept introduction only. |

### 5.3 Buzan Technique Frequency in Aniq Fanlar Sessions

| Phase | Buzan Technique | Frequency |
|---|---|---|
| Phase 0-A (Theme Preview) | BOST Browse + Prime | 100% of sessions |
| Phase 0-B (Flash Cards) | BOST Overview | 100% of sessions |
| Phase 1 (Memory Sprint) | Primacy Effect + Buzan Review Intervals + Link System (occasional) | 100% of sessions |
| Phase 2 (Story Mode) | Radiant Thinking + Dual Coding | 100% of sessions |
| Phase 3 (Game Breaks) | Cortical Diversity (left+right brain balance) + Von Restorff (novelty injection) | 100% of sessions |
| Phase 4 (Real-Life Challenge) | Dual Coding (visual scenario) | 100% of sessions |
| Phase 5 (Consolidation) | Memory Palace + Peg System + 80/20 Keywords + Recency Effect | 100% of sessions |
| Phase 6 (Final Boss) | Von Restorff (visual distinctiveness) + Recency (final review before boss) | 100% of sessions |
| Phase 7 (Reflection) | BOST Review | 100% of sessions |

## 6. BOSS FORMAT RULES FOR ANIQ FANLAR

| Parameter | Grades 1–4 | Grades 5–8 | Grades 9–11 |
|---|---|---|---|
| **Question format** | MC (up to 50%) + short answer | Short answer + open reasoning (MC max 30%, ADR-005) | Open reasoning + extended response only (0% MC) |
| **Partial credit** | Yes — rubric-based | Yes — rubric-based | Yes — rubric-based |
| **Notebook Capture** | Not required | 1 in 3 Boss sessions (student photographs hand-work) | 1 in 2 Boss sessions |
| **Socratic tutoring on failure** | Yes (Tier 3, Premium) | Yes (Tier 3, Premium) | Yes (Tier 3, Premium) |
| **Damage tiers** | Easy (−10 HP, PISA L3), Medium (−20 HP, PISA L4), Hard (−30 HP, PISA L5–6) | Same | Same |
| **Question distribution** | 40% Easy, 40% Medium, 20% Hard | Same | Same |

**Aniq Fanlar Boss Design Principle:** Every Boss question must require the student to SHOW THEIR WORK, not just produce a final answer. For G6+, this means typed step-by-step reasoning or photographed hand-work. A Boss question that can be answered by guessing from multiple-choice options is a failed Boss question.

## 7. PHASE TIME WEIGHT OVERRIDES (ANIQ FANLAR SPECIFIC)

The standard 7-phase session (30–45 min) is weighted differently for Aniq Fanlar:

| Phase | Standard Duration | Aniq Fanlar Duration | Rationale |
|---|---|---|---|
| Phase 1 (Memory Sprint) | 2 min | 2 min | Unchanged — warm-up is equally important across all families. |
| Phase 2 (Story Mode) | 5–7 min | 5–7 min | Unchanged — but CPA staging adds visual/concrete time. |
| Phase 3 (Game Breaks) | 6–9 min | 8–10 min | **+2 min.** Aniq Fanlar requires more practice density. Extra time for Notebook Capture and Error Analysis tasks. |
| Phase 4 (Real-Life Challenge) | 3–5 min | 3–5 min | Unchanged. |
| Phase 5 (Consolidation) | 2–3 min | 3–4 min | **+1 min.** Formula locking via Memory Palace and dual coding requires extra time. |
| Phase 6 (Final Boss) | 5–10 min | 7–12 min | **+2 min.** Open reasoning and hand-work take longer than MC. |
| Phase 7 (Reflection) | 2–3 min | 2–3 min | Unchanged. |
