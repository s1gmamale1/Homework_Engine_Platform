Escape Room: Subject Lock v1

Escape Room is a multi-lock puzzle game. The student must unlock 4 objects in a virtual room by answering subject questions. Solving one lock reveals clue cards for the others.

4 locks Cross-hinting Clue Journal Multi-concept synthesis

Purpose

This game tests multi-concept synthesis, clue linking across tasks, and reasoning under time pressure. Its defining mechanic is cross-hinting: the room is not 4 isolated quizzes.

Core Rules

  • Every room has exactly 4 locked objects: Door, Chest, Window, Drawer.
  • Each lock requires 2-3 correct answers.
  • Solving a lock reveals clue payloads for other locks.
  • Discovered clues are stored in a visible Clue Journal.
  • The student may attempt the locks in any order.
  • The room ends in success only when all 4 locks are solved.
  • If time expires first, the student is locked in.

Core Loop

  1. Show room intro and start timer.
  2. Student chooses any lock.
  3. Lock presents its question set.
  4. Correct answers progress the lock.
  5. Wrong answers cost time and small XP.
  6. Full lock solve reveals clue payloads.
  7. Clues are stored in the journal.
  8. Student uses clues to solve remaining locks.
  9. Escape occurs only when all 4 locks are open.

Clue Journal and Cross-Hinting

The Clue Journal is always visible. It stores discovered clue cards that help with other locks but do not directly give the final answer.

  • One digit of the door code is 6
  • The chest answer starts with A
  • The window pattern begins with the blue tile in the top-right

Without cross-hinting, Escape Room becomes 4 separate quizzes, which is not the intended mechanic.

Timer, Hints, and Failure

  • Easy: 5:00
  • Medium: 4:00
  • Hard: 3:00
  • Elite: 2:30

Hints cost XP and narrow the search space. Wrong answers apply time tax and can trigger an easier variant. Standard rooms allow 2 runs with reset layout or order.

Best Variations

  • Single-Subject Room: all 4 locks use one subject or chapter
  • Cross-Subject Room: each lock uses a different subject
  • Mythical Boss Room: no hints, tighter timer, one attempt, harder multi-subject questions

Buzan Integration

Escape Room is one of the strongest Buzan fits when used properly.

  • Memory Palace: the room itself is the locus structure
  • Link System: each solved lock points toward another
  • Radiant thinking: the student branches across locks
  • TEFCAS: each attempt is trial, feedback, adjust, success
  • Association: object, clue, and concept are remembered together

Anti-Patterns

  • The locks can be solved fully independently
  • Clues directly give away answers
  • All 4 locks feel mechanically identical
  • The timer makes success impossible for the target student
  • The room becomes a trivia quiz instead of a clue network

Acceptance Checklist

  • All 4 locks are clearly distinct
  • The clue journal adds real value
  • Every clue helps another lock without fully solving it
  • The room works in standard time limits
  • The student can choose lock order freely
  • Buzan integration improves memory, clue linkage, or branching reasoning

Canonical Example

Baseline room: You are locked in Al-Khwarizmi's study. The door, chest, window, and drawer are all locked. Each object requires 2-3 correct answers. Every solved object reveals clue cards for the others. Use the clue journal to connect the room and unlock all 4 before time runs out.
