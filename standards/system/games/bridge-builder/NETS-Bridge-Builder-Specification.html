Core Concept

A gap must be crossed. Student places structural elements by answering questions. Each piece costs budget points. Wrong answers waste budget without placing the piece. Build a stable bridge within budget to win.

How to Play

1. Gap Presented     — terrain, width, load requirement
2. Budget Allocated  — fixed points based on difficulty
3. Select Element    — choose where to place beam/cable/support
4. Answer Question   — knowledge gate tied to structural concept
   Correct → element placed, budget deducted
   Wrong   → element NOT placed, budget STILL deducted
5. Repeat            — until bridge spans gap or budget out
6. Load Test         — simulated load crosses bridge
   Stable = WIN | Collapse = FAIL
7. Retry             — Basic: 2 max | Premium: 3 max (+25% budget each retry)

Basic Failure Flow

Attempt 1 → Collapse / budget out
  → "Blueprint Review": shows failed elements + stored feedback
  → Retry with +25% budget
Attempt 2 → Collapse / budget out
  → Game FAILED. Duolingo Mode reviews physics concepts.
  → No more retries this session.

Premium Failure Flow

Attempt 1-2 → Collapse / budget out
  → Tier 3 Socratic: "Which force unbalanced? Where did load path break?"
  → Retry with +25% budget each time
Attempt 3 → Collapse / budget out
  → Game FAILED. Duolingo Mode activates.

Edge Cases

Low-end devices: Skip (high render). Substitute Tile Match / Adaptive Quiz.
AI unavailable: Pre-fetched Tier 1 questions. Hints fall back to stored arrays.
Budget exhausted: Collapse → retry with +25%.
G5 working memory: Max 4-5 elements. Simplify to 3-element. No Peg variant.
Basic 2-attempt + Catch-Up: Still 2 attempts, gap simplified (fewer elements).
Duolingo Mode: Always active both tiers — reviews physics concepts on fail.
