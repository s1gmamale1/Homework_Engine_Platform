Phase 3 Mechanic

NETS Minefield Navigator v1

Minefield Navigator is a knowledge-gated navigation puzzle. The student moves from Start to Exit across a hidden-mine grid, and every move is decided by question performance.

Navigation tension Knowledge-gated control 3 lives 2 runs

Purpose

This game tests content mastery and path planning at the same time. The board creates risk. The questions decide whether the student keeps control of movement.

Core Rules

  • The board is a hidden-mine grid.
  • The student must move from Start to Exit.
  • Every move requires one question.
  • Correct answer: move to the chosen adjacent tile.
  • Wrong answer: the AI forces an adjacent move.
  • If the forced tile contains a mine, one life is lost.
  • The student has 3 lives.
  • Reaching the exit is a win.
  • Losing all 3 lives is a loss.
  • The student gets 2 runs with a new mine layout on retry.

Core Loop

  1. Show Start and Exit on the grid.
  2. Student chooses an adjacent tile.
  3. System asks one question.
  4. Correct answer keeps the chosen move.
  5. Wrong answer triggers a forced adjacent move.
  6. Check whether the new tile is safe or mined.
  7. Update lives and board state.
  8. Repeat until exit or zero lives.

Buzan Integration

Minefield Navigator should use Buzan-aligned elements only where they improve navigation, recall, or transfer.

  • Memory Palace logic: the grid can act as a simple spatial memory board.
  • Association Effect: safe routes are easier to remember when zones have meaningful anchors.
  • Radiant thinking: branching route choice should reward comparing multiple paths.
  • BOST-style review feel: feedback after a wrong move should reconnect the concept to the location.

Avoid decorative Buzan theming that does not improve memory, route choice, or concept linkage.

Best Variations

  • Coordinate Navigator: math grid movement
  • Map Route Navigator: geography-style terrain crossing
  • Timeline Navigator: movement through ordered historical milestones
  • Cross-Subject Hazard Grid: rows or zones use different question pools

Anti-Patterns

  • Movement stays random even after correct answers
  • The student can win by pure luck
  • The forced move is always instant death
  • The grid is decorative only
  • The questions have no relation to the board metaphor
  • Buzan-themed elements are pasted on without improving memory or navigation
  • Question formats are too slow for turn-by-turn play

Acceptance Checklist

  • Correct answers preserve player control
  • Wrong answers remove control but do not make loss automatic
  • The board offers meaningful route choice
  • The mine layout creates tension without unfair traps
  • The subject metaphor fits the movement system
  • Any Buzan-style spatial anchors improve recall or route reasoning
  • The question format matches fast navigation pacing
  • Basic and Premium rules are clearly separated
  • The game works in two short runs

Story Mode Note

For future Story Mode work, Buzan integration should be explicit where relevant: narrative as memory scaffold, strong association hooks, clean chunking, and visual or radiant structure where it improves recall.

Story Mode should build memorable structure, and Minefield can later reuse that structure spatially.

Canonical Example

Good baseline: You are navigating a 5x5 hazard grid from Start to Exit. You choose one adjacent tile each turn. If you answer correctly, you move there. If you answer wrong, the AI forces an adjacent move. Some tiles are mined. You have 3 lives. Reach the exit before losing all lives.
